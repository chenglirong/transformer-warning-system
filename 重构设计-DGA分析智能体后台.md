## 题目(已定)

> **变压器油中溶解气体分析智能体研究与实现**

- 「分析」取 DGA(Dissolved Gas Analysis)本义,对标 DL/T 722《变压器油中溶解气体**分析和判断**导则》;指「检测 + 故障类型判断 + 措施建议 + 趋势分析」全流程 —— 把行业标准串起来输出分析报告。DL/T 722 第1章职责即「故障识别 + 故障类型判断 + 建议进一步采取的措施」,故报告**依国标规则法(三比值/Duval/特征气体)直接给出故障类型判断结论 + 措施建议**,不自缚为「仅供参考/不确诊」。「智能体」= Agent 编排(创新点)。

## 一、系统定位(一句话)

> **面向 220kV 及以下油浸式变压器,基于 DGA 的在线监测后台软件:做异常检测、故障类型判断(三比值/Duval/特征气体)、分级预警/告警、措施建议,并基于产气速率做趋势预警;创新点在于用智能体(Agent)编排整个分析—预警流程。**

定位依据(国标原文):

- DL/T 1498.2-2025 **§4.7**:后台具备**分析、诊断、展示**功能 —— 你做的就是这个后台软件。
- DL/T 249-2025 §3.1:监测 7 种气体(H₂/CH₄/C₂H₄/C₂H₆/C₂H₂/CO/CO₂)= 「多组分」在线监测装置 —— 对应本系统。
  - **气体口径(已定,7 种全用,按国标分工)**:DGA 标准含 7 种特征气体(DL/T 722 §3.1),合成与报告**7 种全生成、全展示**,不再分「主/辅」。但**不是处处平等参与同一判据**——按国标该用之处用:
    - **合成时 7 种全生成有值**(见 §六):CO/CO₂ 的合成分布锚点取自原始 xlsx 有值部分(CO 78%/CO₂ 69%)自行统计,可靠性弱于烃类(Analysis 页无 CO/CO₂ 统计),答辩如实交代;但合成时序**每天 7 种都有值,主流程零缺值,不需要填充/降可信度逻辑**。
    - **表A.3 四档落档**只吃乙炔 / H₂ / 总烃(浓度·增量·总烃周增率);**CO/CO₂ 不进表A.3 阈值**。
    - **三比值 / Duval**:运算底座为本就只吃的 5 烃类 + H₂;总烃、722 产气速率同用烃类系。
    - **CO/CO₂**:特征气体法(表5)区分「涉油 vs 涉油+纸」的主要/次要特征气体(§10.1 表5 原文)、§10.2.3.1 CO₂/CO 比值判固体绝缘老化;表5 里油纸过热/局放/电弧等类的**主判据含 CO**,不是「辅气」。
    - ⚠️ **纠正原「5主2辅、CO/CO₂ 仅辅助」旧表述**:勿贬辅;亦勿写成「7 气处处同等进所有门槛」——分工如上。

## 二、为什么重构成「检测+故障类型判断+告警」为主、预测为辅

两个硬约束决定的:

1. **数据约束**:`bmrayan` 是 **743 条无时间戳快照**(已核实无日期列)→ 做不了时序预测,但适合检测/判断。
2. **标准约束**:DL/T 722 整篇是判断方法;DL/T 1498.2 §5.4.2 要求后台提供三比值法 +「大卫三角形法**或**立体图示法」、§5.4.3+附录A 要求告警 → 检测/判断/告警是国标钦定的后台功能。

→ 结论:**主线 = 检测 + 故障类型判断 + 告警(成链);辅线 = 产气速率趋势(服务告警,不做预测模型)。**

## 四、模块设计(主线两大块 + 辅线)

```
原始743快照(分布原料)→[合成360天时序]→[预处理洗净]→ Agent 对每天:
   ①检测=分级(表A.3四档全报)+ 处置紧急度研判(产气速率§9.3.3,判急不急)
   ②故障类型判断(三比值/Duval/特征气体 多方融合)—— 注意值2及以上 **或** 722 相对产气速率超注意值时触发(§10.3 + §10.2.4 a)
   ③趋势(产气速率展示 + 「预」提前预警)
   → 输出【四档档位】+【处置紧急度研判】+【故障类型判断+措施建议】+【趋势】
```

> 全程跑合成时序(检测含产气速率维度需时间,故不分原始/合成两支);原始743 仅作合成分布原料 + 答辩交代。见 §六数据分工总纲。

### 模块0 数据预处理(进异常分析之前)

> ⚠️ **清洗对象 = 原始 743 快照(仅作合成分布锚点用)**,不是主流程数据。主流程跑的合成时序 7 种全生成有值、无缺失,不经过缺失处理。本节 3 步只用于「从原始 743 提取合成锚点」这一步。数据较干净(**5 种烃类 float 零缺失**,CO/CO₂ 含 `'-'`),**不需要堆一长串清洗方法**。

**必做(3 步,洗原始 743 供提锚点)**:

1. **类型清洗**:CO/CO₂ 列里的 `'-'`(代表未测)→ NaN,列自动恢复数值类型(CO/CO₂ 原是 object,唯一非数值就是 `'-'`)。
2. **选列**:取 7 种特征气体(H₂/CH₄/C₂H₄/C₂H₆/C₂H₂/CO/CO₂);弃 H₂O(整列全空)/O₂/N₂(非故障气体)。
3. **缺失处理**:原始 743 中 CO 缺约 22%、CO₂ 缺约 31%、C₂H₂ 缺 1。**提合成锚点时用有值部分统计即可(不填充原始快照)**;合成时序则 7 种全生成有值。

**按需可选(默认不做,触发条件写明)**:

- **IQR 箱线图剔异常**:仅当探查发现明显坏点时才做。⚠️ 数据量小(743)且要保留真实高浓度故障样本,贸然 IQR 易误删早期故障小幅抬升,故**默认只探查不剔除**。
- **对数变换降偏**:**默认不做**。全系统方法均为国标规则(表A.3 看绝对阈值、产气速率看变化率、三比值/Duval 看比值),均不吃绝对偏态,无降偏动机;仅在探查中确有分析需要时再加。

**造特征(全程在合成360天上)**:气体比值(三比值)+ 总烃 + Duval 三角坐标(供分级/诊断)+ 产气速率(一阶差分,供检测涨势研判与模块3趋势)。沿用现有 `features.py`,但**删滑窗统计(不做 LSTM)、删油温速率(无油温字段)、删 MinMax 归一化(原供孤立森林,已去 ML)**。

### 模块1 异常分析(主线核心 = 检测 + 故障类型判断,同一模块分两步)

> 检测与故障类型判断是同一条链上的两步:检测答「在哪一档 + 处置急不急」,故障类型判断答「是何种故障类型」(依国标规则法给结论)。国标也分列(§5.4.2 判断方法 vs 检测/告警),勿混为「检测=判类型」。

**第①步 检测 = 分级(主)+ 处置紧急度研判(路A,已定)**

- **分级(主,套国标表A.3)**:对每条数据按 DL/T 1498.2 附录A 表A.3 落到**四档**——正常 / 注意值1 / 注意值2 / 告警值,**四档全报**(正常也输出「正常,各气体在范围内」)。**正常↔注意值1 的区分纯靠表A.3**(原阈值法并入此处,不再单列——阈值法与表A.3 同为「看绝对值阈值」,单列会重复)。
- **⚠️ 应用原则 ≠ 改档位(DL/T 722 §9.3.3,关键理解)**:表A.3 分级是**事实陈述**(超了就是超了,如实四档全报,不被推翻);§9.3.3 应用原则调的是**「紧急程度/处置建议」,不是改档位**。这些解读放 **Agent C(处置建议)**,不动分级结果:
  - **a) 超注意值但长期稳定** → 档位仍如实报(如注意值2),但处置建议=「长期稳定,加强监视即可,不紧急」(§9.3.3 a「可继续运行」)。
  - **浓度未超但产气速率超** → 按 §9.3.3 a) 缩短采集周期 = **涨势预警**。**这批日子 = 题目的「预」**。落档看表A.3(%/周);「预」看 722(%/月),两套不换算。总烃≥10μL/L 才启用相对速率判据。判型侧另依 §10.2.4 a / §10.3 a:**增长率注意亦可作为进判型前提**(见第②步触发时机)。
  - **d) 仅 H₂ 超但不涨**(§9.3.3 d):722 原文「也可判断为正常」;对接表A.3 四档时**档位仍如实报**,处置降紧急度(「加强监视」)——**协调设计,非 722 原文口径**。
  - **c) 产气速率突然增长**(§9.3.3 c) → 喂 Agent C 作决策触发。
  - **e) 非故障气体来源提示**(§9.3.3 e + §4.3):§4.3 列了 7 种非故障产气来源(不锈钢催化生 H₂、电焊、阳光照射、有载调压开关油污染渗漏带 C₂H₂、油中含水与铁作用生 H₂ 等)。异常档位时,报告加一行「注意核查是否存在非故障气体来源(如有载调压开关油污染、不锈钢催化、设备近期大修等),参 DL/T 722 §4.3」——不改档位,只提示核查。
  - ❌ **不用 §9.3.3 b)**(330kV 首检 C₂H₂≥0.1):本系统 220kV 及以下,电压不匹配。
- **处置紧急度研判(仅注意值2及以上启动)**:用 **DL/T 722 §9.3.2 相对产气速率(%/月)** 判「这个异常急不急、要不要动作」,供 Agent C 处置决策,**不改分级档位**(档位由表A.3 定、是事实)。依据 §9.3.3 a) 前半:「超过注意值时**结合产气速率**判断」——此处「注意值」= 表3 = 表A.3 注意值2,故**只在注意值2+ 启动研判**。合成无油量算不了绝对式1,用相对式2(总烃 10%/月);逐日取月环比(今 vs 30天前);当日超注意值即认定(不加自造连续确认窗,D-011)。
  - ⚠️ **门槛分离(有意为之)**:判型 = **注意值2+ 或** 722 相对产气速率超注意(含涨势预警,§10.2.4 a / §10.3 a);研判(处置紧急度) = **仅注意值2+**(§9.3.3 a 前半)。涨势预警 → 缩周期 + 趋势监视 + **进判型**,但仍**不做**紧急度研判(急缓只在注意值2+评)。
  - **只用产气速率单视角(去机器学习)**:锚定 §9.3.3 a) 一条国标判据,最诚实。原孤立森林属 ML、非国标,与「全程国标背书」主线冲突(同 OCSVM 之理已砍)。多方法交叉一致性收敛到**故障类型判断层**(三方正交有真实价值),检测层不造伪多视角。
  - **两套速率各归各(D-004,关键别混)**:系统有两套口径不同、不可换算的速率——
    - **(A)1498.2 表A.3 ②③(%/周·参比A.3.3 前14~7天窗均值,两端含)** → **只管① 落档**(与浓度并列取最高档,速率超能把档位顶到注意值1/2)。
    - **(B)722 §9.3.2 相对速率(%/月·月环比)** → 管 **② 处置研判**(注意值2+ 判急不急,§9.3.3)、**③ 趋势展示**(模块3)、**④「预」**(浓度未超但相对速率超10%/月→缩周期)。
    - 为什么 ②③④ 归 B 不归 A:§9.3.3「结合产气速率」配套的是 §9.3.2 那套(同一份 722),不是 1498.2 表A.3。两套周期(周/月)、参比(窗均值/月环比)、门槛全不同,报告分别标清。
- **产出**:四档档位(事实)+ (注意值2以上)产气速率处置紧急度研判 + 哪些气体超标 + (Agent C)处置建议。**不算 F1**(四档由国标定,不打分;处置研判靠产气速率判据,诊断可信度靠三方一致性)。

**第②步 故障类型判断(答:是何种故障类型)——多方融合,全部国标规则法**

> ⚠️ **触发时机**(对齐 §10.2.4 a / §10.3 a):**注意值2及以上**(表3 = 表A.3 注意值2)**或** 722 相对产气速率超注意(含涨势预警:档仍正常/注意值1)均可进入判型。比值法须在「含量注意值 **或** 增长率注意值」有理由怀疑故障时才有效;§10.3 a 亦允许短期内迅速增长、尚未超表3 时先判内部异常再进类型判断。涨势预警同时仍走 §9.3.3 a 缩周期。
>
> ⚠️ **低浓度局限(§10.2.4 c)**:浓度<10μL/L 时比值法误差大,标**可信度低**(仍给结论)。

> 多方融合:三比值 + Duval + 特征气体。§5.4.2 要求三比值 +「大卫三角**或**立体图示」——本系统选大卫三角。

采用的国标方法(三种,做多方法判定对比):

1. **特征气体法(关键气体法)** —— DL/T 722 §10.1 表5(原文逐格,实现照此)
   - | 故障类型           | 主要特征气体       | 次要特征气体     |
     | ------------------ | ------------------ | ---------------- |
     | 油过热             | CH₄、C₂H₄          | H₂、C₂H₆         |
     | 油和纸过热         | CH₄、C₂H₄、CO      | H₂、C₂H₆、CO₂    |
     | 油纸绝缘中局部放电 | H₂、CH₄、CO        | C₂H₄、C₂H₆、C₂H₂ |
     | 油中火花放电       | H₂、C₂H₂           | —                |
     | 油中电弧           | H₂、C₂H₂、C₂H₄     | CH₄、C₂H₆        |
     | 油和纸中电弧       | H₂、C₂H₂、C₂H₄、CO | CH₄、C₂H₆、CO₂   |
   - 实现:看哪些气体偏高 → 匹配本表给判定。**表5 本身无「偏高」数值与加权分**;本系统工程实现见 `key_gas.py`(D-018):有表3处贴 220kV 注意值(H₂/C₂H₂/总烃),其余门槛与乘子为工程经验,答辩摊牌不装作国标数字。
2. **三比值法** —— DL/T 722 §10.2.1 表6/表7(已实现,`ratios.py`;实现照下表原文)
   - 三对比值(C₂H₂/C₂H₄、CH₄/H₂、C₂H₄/C₂H₆)编码→判定。**属 DL/T 722**(在 IEC 60599 基础上按国内经验细化)。
   - **表6 编码规则(原文逐格)**:
     | 气体比值范围 | C₂H₂/C₂H₄ | CH₄/H₂ | C₂H₄/C₂H₆ |
     |---|---|---|---|
     | <0.1 | 0 | 1 | 0 |
     | [0.1,1) | 1 | 0 | 0 |
     | [1,3) | 1 | 2 | 1 |
     | ≥3 | 2 | 2 | 2 |
   - **表7 故障类型判断(原文逐格)**:
     | C₂H₂/C₂H₄ | CH₄/H₂ | C₂H₄/C₂H₆ | 故障类型判断 |
     |---|---|---|---|
     | 0 | 0 | 1 | 低温过热(低于150℃) |
     | 0 | 2 | 0 | 低温过热(150℃~300℃) |
     | 0 | 2 | 1 | 中温过热(300℃~700℃) |
     | 0 | 0,1,2 | 2 | 高温过热(高于700℃) |
     | 0 | 1 | 0 | 局部放电 |
     | 2 | 0,1 | 0,1,2 | 低能放电 |
     | 2 | 2 | 0,1,2 | 低能放电兼过热 |
     | 1 | 0,1 | 0,1,2 | 电弧放电 |
     | 1 | 2 | 0,1,2 | 电弧放电兼过热 |
3. **大卫三角形法** —— DL/T 722 附录C 图C.2 + 表C.1

**三方法怎么统一/比一致性(已定,关键)**:三方法分类粒度不同,**不强行统一代码**(国标无表5↔六代码官方映射,硬映射=无依据瞎输出),而是分两层看一致性:

**增值 / 答辩点**:分类粒度不同不是缺陷,恰体现「多方法各有侧重」(三比值/Duval 给六分区细分,特征气体法补「油/纸」维度并以附注进结论);零编造、全程国标原文。输出示例:「判定高温过热(高温 T3),三比值与大卫三角一致、特征气体法亦指向油过热——三方一致,可信度高。表5提示可能涉及固体绝缘（油+纸）。建议结合附录D推荐的电气试验(绕组直流电阻、铁芯绝缘电阻等)进一步核实故障部位。」

    1. **大类级(722 附录D 表D.1,一定报)**:按故障大类给整列试验——过热性故障→绕组直流电阻/铁芯绝缘电阻及接地电流/空载损耗/绝缘油糖醛等;放电性故障→绕组直流电阻/局部放电测量/绝缘油击穿电压·含水量/工频耐压等。这是 722 判出故障后国标写死的推荐项目,与趋势无关,只按过热/放电分。
    2. **细化级(1685 附录B 表B.2/B.3,可选)**:匹配状态量描述时补报第二列试验(不碰第三列成因):

       - B.2② 同气体描述 → 铁芯接地电流 + 铁芯绝缘电阻;
       - B.2④「C₂H₄、CO、CO₂ 增长较快」→ 分相低电压短路损耗;
       - B.2⑥「C₂H₆、C₂H₄ 较快,有时 H₂/C₂H₂」→ 红外测温查套管接头;
       - B.2⑦「色谱高能放电特征、乙炔增长快」→ 有载开关油位/储油柜对照;
       - B.2⑧/⑨「色谱高温过热、总烃增长快」→ 直阻稳定性检查 / 1.1 倍过电流跟踪产气;
       - B.3②「C₂H₂ 单项增高 + 油中带电度超标」→ 静电感应电压或泄流电流;
       - B.3⑥「以 C₂H₂ 为主,通常 C₂H₄ 比 CH₄ 低」→ 超声波随负荷对照。
    - ⚠️ 只荐试验、不下成因;不做 1685 附录A 扣分评价。
  - ⚠️ 可信度:低浓度<10μL/L 或三方打架 → 标低,仍给结论。
  - ❌ 不做:健康评分(1685 扣分 / 984 老化)、运维处置指令(只到附录D 试验层)。
  - 依据:DL/T 722 第1章职责 = 「故障识别 + 故障类型判断 + 建议进一步采取的措施」——判类型、给措施正是国标要求。
- **用国标规则法而非 ML 多分类器(数据事实,诚实交代)**:三比值/Duval/特征气体是**规则查表法,不需训练、不需标签**,直接给类型结论完全合规。之所以不额外训 ML 多分类器,是因为 `bmrayan` 真故障仅 4 条(18 条标签),验证不了 ML 多分类准确率 —— 故用国标规则法。措施建议依据:**DL/T 722 附录D 表D.1「据 DGA 判断有故障后,结合表D.1 推荐的其他电气试验(绕组直流电阻/铁芯绝缘/绝缘油击穿/局放测量等,多需停电)才有助于故障部位判断」**——即 **DGA 后台的职责边界到「故障类型判断(过热/放电性质 + 六分区)」这层**,报告如实给出此结论 + 附录D 措施建议;而**具体成因/物理部位定位**(哪个绕组、哪处接头)需接停电电气试验,超出在线 DGA 能力,如实标注「部位定位需附录D 停电试验」。这不是「不敢确诊类型」,而是「类型已判定,部位待现场试验」——层次分清:**类型判断=系统输出,部位定位=展望**。

### 模块3 趋势预警(辅线——722 §9.3.2 式2 月度产气速率展示)

- **职责**:基于合成时序算 **DL/T 722 §9.3.2 式2 相对产气速率(%/月)**,做趋势曲线展示 + 总烃 10%/月 判据旁证。与② 处置研判、④「预」**同一套速率**(722 §9.3.2,%/月),只是本模块做**月度整体展示**(相邻两月均值),②④在检测里做**逐日**(月环比,当日超注意即认定)。
- **⚠️ 两套速率各归各(D-004,别混)**:系统里有**两套口径不同、不可换算**的速率——(A)**1498.2 表A.3 ②③(%/周,参比A.3.3 前14~7天窗均值)**:在 grade.py 里做**落档**(①);(B)**722 §9.3.2 相对速率(%/月)**:做 **②处置研判 + ③趋势展示 + ④「预」**(§9.3.3 配套的产气速率就是同一份 722 的 §9.3.2)。两套周期(周 vs 月)/参比(窗均值 vs 月环比)/门槛全不同,不统一、不互相换算,报告里分别标清。

- **国标计算公式(DL/T 722 §9.3.2,实现照此)**:
  - **相对产气速率(式2,推荐——只需前后两次浓度,最易实现)**:
    γᵣ = (C₂ − C₁) / C₁ × 1/Δt × 100% (%/月);C₂=后次浓度,C₁=前次浓度,Δt=间隔(月)
  - 绝对产气速率(式1,需油量m、油密度ρ,较繁):γₐ = (C₂−C₁)/Δt × m/ρ (mL/天)
  - **总烃相对产气速率注意值 = 10%/月**(总烃起始含量很低<10μL/L 时不宜用此判据,故设「总烃≥10μL/L」门槛,依据 §9.3.2)。
  - 表4 绝对产气速率注意值(mL/d,密封式/开放式,**原图核对**):H₂ 10/5、C₂H₂ 0.2/0.1、总烃 12/6、CO 100/50、CO₂ 200/100。
    - 注:C₂H₂<0.1 且总烃低时总烃产气率可不分析;检测周期缩短时本表仅供参考。

- **「预」的国标合法性依据(关键,答辩引用)** —— DL/T 722 §9.3.3 a):

  > 「气体含量虽**低于注意值**,但**产气速率超过注意值**,也应缩短检测周期(预警)。」
  > → 这就是「**预**」:浓度还没超,但速率已超 → ① 表A.3 落档把档位顶到注意值1/2(档位是事实)+ ② §9.3.3 a) 缩短检测周期(提前预警)。不是靠预测模型,靠产气速率判据,且**有国标明文撑腰**。

- **不做时序预测模型**(ARIMA/LSTM 等一律不做):辅线只做产气速率/增量的**趋势展示 + 判据预警**,不做未来浓度预测。合成数据上论证趋势判据可行即可,不吹精度。

## 六、数据策略(已定:bmrayan 真实快照 + 合成时序)

### 数据分工总纲(地基,据物理事实定死)

> **Agent 全程跑合成 360 天时序**,对每天:分级(表A.3 四档全报)+ 检测处置紧急度研判(产气速率 §9.3.3)+ 故障类型判断 + 趋势。

**为什么全程用合成(不再分原始/合成两支)**:检测的「处置紧急度研判」含**产气速率(涨势)**维度,而产气速率**需要时间序列**(算前后浓度差),原始 743 快照无时间戳算不了 → 检测既然要算涨势,就必须跑在合成时序上。故**统一在合成 360 天上跑全链路**,同一台虚拟设备贯通,Agent 串得通。

**不算 F1、不需要标签**:四档由国标表A.3 直接定(不打分);检测处置研判靠产气速率(§9.3.3,不依赖 ground truth),诊断可信度靠三比值/Duval/特征气体三方一致性。故**不纠结原始标签/F1**。

**原始 743 的角色**:① 合成的**分布原料**(按其真实分布——偏度/相关性合成,见下「合成锚点」);② 答辩**诚实交代**数据来源。**不直接进主流程跑检测。**

- **原始数据**:`bmrayan` 743 条真实快照(数据集官方定义 **5 种烃类 100% 完整**;CO 78%/CO₂ 69% 有值,作合成锚点;18条故障标签;真实 μL/L 量级)。
- **合成时序(关键)**:原始快照**无时间戳**,为获得时间维度,**基于其真实统计分布合成「单台虚拟变压器 × 360 天」时序数据**(保持原始分布特性:偏度、气体相关性等,bmrayan Analysis 页有真实统计值可引用)。
  - **规模(已定)**:单台 × 360 天,健康约 75% / 异常约 25%,跨完整四季。单台最干净好讲(原始无设备标识,多台合成更假);360 天够算产气速率/趋势、跨四季,再多属注水。(注:合成时序服务**全链路**——检测/诊断/趋势都在合成上跑,因检测含产气速率维度需时间,见上「数据分工总纲」。)
  - **合成对齐的真实统计锚点(供合成趋势数据用,取自 bmrayan Analysis 页,答辩证据——合成非瞎编)**:
    - **分布形态**:所有气体呈**对数正态、强右偏**(偏度 7.76~16.94,C₂H₄ 最高 16.94),跨 4 个数量级。→ 合成应按对数正态生成(锚点取自 bmrayan Analysis 页真实统计值)。
    - **量级锚点**:H₂ 均值 348.4 ppm、范围 0~20000+;C₂H₄ 最大 40000+。
    - **相关性矩阵(合成须保持)**:CH₄-C₂H₆ 0.846(最强)、CH₄-C₂H₄ 0.752、C₂H₆-C₂H₄ 0.698、H₂-CH₄ 0.521、H₂-C₂H₂ 0.357(弱)。→ 烃类强相关(0.7~0.85)、H₂/C₂H₂ 相对独立。

  - ⚠️ **CO/CO₂ 缺口**:Analysis 页**只统计了 5 种烃类气体,无 CO/CO₂ 的分布/相关性**。合成 CO/CO₂ 时序时无 Analysis 锚点,只能用 xlsx 原始列(CO 78%/CO₂ 69% 有值)自行统计,可靠性弱于烃类——答辩如实交代。

- **故障类型多分类** → 原始仅 18 条标签,验证不了,**留作展望**,不作主线主张。
- **诚实声明(P1 原则,答辩必讲)**:
  1. 时序数据是**合成的**(原始无时间戳),论文如实标注合成方法与依据;
