## 题目(已定)

> **变压器油中溶解气体分析智能体研究与实现**

- 「分析」取 DGA(Dissolved Gas Analysis)本义,对标 DL/T 722《变压器油中溶解气体**分析和判断**导则》;指「检测 + 故障类型判断 + 措施建议 + 趋势分析」全流程 —— 把行业标准串起来输出分析报告。DL/T 722 第1章职责即「故障识别 + 故障类型判断 + 建议进一步采取的措施」,故报告**依国标规则法(三比值/Duval/特征气体)直接给出故障类型判断结论 + 措施建议**,不自缚为「仅供参考/不确诊」。「智能体」= Agent 编排(创新点)。

---

## 一、系统定位(一句话)

> **面向 220kV 及以下油浸式变压器,基于 DGA 的在线监测后台软件:做异常检测、故障类型判断(三比值/Duval/特征气体)、分级预警/告警、措施建议,并基于产气速率做趋势预警;创新点在于用智能体(Agent)编排整个分析—预警流程。**

定位依据(国标原文):
- DL/T 1498.2-2025 §4.5 / DL/T 249-2025 §4.5:在线监测装置的「站端后台」应具备**分析、诊断、展示**功能 —— 你做的就是这个后台软件。
- DL/T 249-2025 §3.1:监测 7 种气体(H₂/CH₄/C₂H₄/C₂H₆/C₂H₂/CO/CO₂)= 「多组分」在线监测装置 —— 对应本系统。
  - **气体口径(已定,5主+2辅)**:DGA 标准含 7 种气体,但本数据集 **5 种烃类(H₂/CH₄/C₂H₄/C₂H₆/C₂H₂)100% 完整,作检测/诊断核心**;**CO/CO₂ 部分可得(78%/69%),仅作辅助参考**(特征气体法区分「油 vs 油+纸」时用),因残缺**不纳入核心判据与合成**——如实交代。实测核心方法(表A.3 分级、三比值法、Duval)本就只用 5 种烃类+H₂,不依赖 CO/CO₂。

## 二、为什么重构成「检测+故障类型判断+告警」为主、预测为辅

两个硬约束决定的:

1. **数据约束**:`bmrayan` 是 **743 条无时间戳快照**(已核实无日期列)→ 做不了时序预测,但适合检测/判断。
2. **标准约束**:DL/T 722 整篇是判断方法;DL/T 1498.2 §5.4.2 要求后台提供「三比值法、大卫三角」、§5.4.3+附录A 要求告警 → 检测/判断/告警是国标钦定的后台功能。

→ 结论:**主线 = 检测 + 故障类型判断 + 告警(成链);辅线 = 产气速率趋势(服务告警,不做预测模型)。**

## 四、模块设计(主线两大块 + 辅线)

```
原始743快照(分布原料)→[合成360天时序]→[预处理洗净]→ Agent 对每天:
   ①检测=分级(表A.3四档全报)+ 处置紧急度研判(产气速率§9.3.3,判急不急)
   ②故障类型判断(三比值/Duval/特征气体 多方融合)—— 仅注意值2及以上触发(§10.3先比表3=注意值2→才判类型)
   ③趋势(产气速率展示 + 「预」提前预警)
   → 输出【四档档位】+【处置紧急度研判】+【故障类型判断+措施建议】+【趋势】
```
> 全程跑合成时序(检测含产气速率维度需时间,故不分原始/合成两支);原始743 仅作合成分布原料 + 答辩交代。见 §六数据分工总纲。

### 模块0 数据预处理(进异常分析之前)

> 数据较干净(**5 种烃类 float 零缺失**,CO/CO₂ 残缺作辅助),**不需要堆一长串清洗方法**。真正必做的只有「类型清洗 + 选列 + 缺失标注」3 步;IQR 剔异常、对数变换列为**按需可选**,默认不做。

**必做(3 步,洗数据)**:
1. **类型清洗**:CO/CO₂ 列里的 `'-'`(代表未测)→ NaN,列自动恢复数值类型(CO/CO₂ 原是 object,唯一非数值就是 `'-'`)。
2. **选列**:核心取 **5 种烃类(H₂/CH₄/C₂H₄/C₂H₆/C₂H₂)**,CO/CO₂ 作辅助列保留(残缺);弃 H₂O(整列全空)/O₂/N₂(非故障气体)。
3. **缺失标注**:CO 缺约 22%(78% 有值)、CO₂ 缺约 31%(69% 有值)、C₂H₂ 缺 1。**不强行填满**,标注完整度;该列有值则用、无值则跳过(诚实原则)。如需填补,中位数填充即可,不上 KNN。

**按需可选(默认不做,触发条件写明)**:
- **IQR 箱线图剔异常**:仅当探查发现明显坏点时才做。⚠️ 数据量小(743)且要保留真实高浓度故障样本,贸然 IQR 易误删早期故障小幅抬升,故**默认只探查不剔除**。
- **对数变换降偏**:**默认不做**。全系统方法均为国标规则(表A.3 看绝对阈值、产气速率看变化率、三比值/Duval 看比值),均不吃绝对偏态,无降偏动机;仅在探查中确有分析需要时再加。

**造特征(全程在合成360天上)**:气体比值(三比值)+ 总烃 + Duval 三角坐标(供分级/诊断)+ 产气速率(一阶差分,供检测涨势研判与模块3趋势)。沿用现有 `features.py`,但**删滑窗统计(不做 LSTM)、删油温速率(无油温字段)、删 MinMax 归一化(原供孤立森林,已去 ML)**。

### 模块1 异常分析(主线核心 = 检测 + 故障类型判断,同一模块分两步)

> 检测与故障类型判断是同一条链上的两步:检测答「在哪一档 + 处置急不急」,故障类型判断答「是何种故障类型」(依国标规则法给结论)。国标也分列(§5.4.2 判断方法 vs 检测/告警),勿混为「检测=判类型」。

**第①步 检测 = 分级(主)+ 处置紧急度研判(路A,已定)**

- **分级(主,套国标表A.3)**:对每条数据按 DL/T 1498.2 附录A 表A.3 落到**四档**——正常 / 注意值1 / 注意值2 / 告警值,**四档全报**(正常也输出「正常,各气体在范围内」)。**正常↔注意值1 的区分纯靠表A.3**(原阈值法并入此处,不再单列——阈值法与表A.3 同为「看绝对值阈值」,单列会重复)。
- **⚠️ 应用原则 ≠ 改档位(DL/T 722 §9.3.3,关键理解)**:表A.3 分级是**事实陈述**(超了就是超了,如实四档全报,不被推翻);§9.3.3 应用原则调的是**「紧急程度/处置建议」,不是改档位**。这些解读放 **Agent C(处置建议)**,不动分级结果:
  - **a) 超注意值但长期稳定** → 档位仍如实报(如注意值2),但处置建议=「长期稳定,加强监视即可,不紧急」(§9.3.3 a「可继续运行」)。
  - **未超注意值但产气速率超** → 档位仍报「正常/注意值1」,但处置建议=「涨势快,建议缩短周期」(即「预」,§9.3.3 a)。
  - **d) 仅 H₂ 超但不涨**(§9.3.3 d):H₂ 影响因素多(§4.3),**档位如实报(超了就报注意值),但处置解读为「H₂ 单项偏高且稳定,大概率非故障,加强监视」** —— 不改档位、只降紧急度。(⚠️ 早先「判正常」表述已纠正:不推翻档位,否则与分级自相矛盾。)
  - **c) 产气速率突然增长**(§9.3.3 c) → 喂 Agent C 作决策触发。
  - ❌ **不用 §9.3.3 b)**(330kV 首检 C₂H₂≥0.1):本系统 220kV 及以下,电压不匹配。
- **处置紧急度研判(仅注意值2及以上启动)**:用**产气速率视角**判「这个异常急不急、要不要动作」,供 Agent C 处置决策,**不改分级档位**(档位由表A.3 定、是事实)。依据 §9.3.3 a):「超过注意值时结合产气速率判断;超过但长期稳定可继续运行」——国标要的是「超了之后据产气速率定处置」,不是「再确认一遍超没超」。⚠️ **注意值2(C₂H₂≥5/H₂≥150/总烃≥150)= 722 表3 经典注意值线**(表A.3 注意值2 与 722 表3 同一条线),故门槛卡注意值2 有国标撑腰。
  - **只用产气速率单视角(去机器学习)**:锚定 §9.3.3 a) 一条国标判据,最诚实。原孤立森林属 ML、非国标,与「全程国标背书」主线冲突(同 OCSVM 之理已砍)。多方法交叉一致性收敛到**故障类型判断层**(三方正交有真实价值),检测层不造伪多视角。
  - 产气速率两处分工:**此处**判涨势超没超注意值→供处置研判;**模块3**做完整趋势展示 + 「预」提前预警。
- **产出**:四档档位(事实)+ (注意值2以上)产气速率处置紧急度研判 + 哪些气体超标 + (Agent C)处置建议。**不算 F1**(四档由国标定,不打分;处置研判靠产气速率判据,诊断可信度靠三方一致性)。

**第②步 故障类型判断(答:是何种故障类型)——多方融合,全部国标规则法**

> ⚠️ **触发时机(§10.3 + §10.2.4 a,别每条都跑)**:§10.3 判断步骤——先比表3注意值(认为有故障)→ 才用表5/6/7 判类型。**表3注意值 = 表A.3 注意值2**(同一条线),故**故障类型判断仅对注意值2及以上触发**(与处置研判同门槛);注意值1 及正常档只报档位、不判类型。§10.2.4 a) 亦印证:比值法只在超注意值后才有效。
>
> ⚠️ **低浓度局限(§10.2.4 c)**:浓度<10μL/L 时比值法误差大(>10μL/L 两次测试误差≤10%,比值误差已达20%),故低浓度样本的判断结论**标可信度低**(仍给结论)——诚实的局限,答辩加分。

> 与检测的「处置紧急度研判」对称:故障类型判断也是**多方融合**(三比值 + Duval + 特征气体法 三方),看判定是否一致——「多方法融合」是贯穿全系统的方法论主线。纳入 1498.2 §5.4.2 **点名要求**的三比值法 + 大卫三角形法,全程国标背书,不引深度学习(轻量)。

采用的国标方法(三种,做多方法判定对比):

1. **特征气体法(关键气体法)** —— DL/T 722 §10.1 表5(原文逐格,实现照此)
   - | 故障类型 | 主要特征气体 | 次要特征气体 |
     |---|---|---|
     | 油过热 | CH₄、C₂H₄ | H₂、C₂H₆ |
     | 油和纸过热 | CH₄、C₂H₄、CO | H₂、C₂H₆、CO₂ |
     | 油纸绝缘中局部放电 | H₂、CH₄、CO | C₂H₄、C₂H₆、C₂H₂ |
     | 油中火花放电 | H₂、C₂H₂ | — |
     | 油中电弧 | H₂、C₂H₂、C₂H₄ | CH₄、C₂H₆ |
     | 油和纸中电弧 | H₂、C₂H₂、C₂H₄、CO | CH₄、C₂H₆、CO₂ |
   - 实现:看哪些气体偏高 → 匹配本表给判定。简单、好讲。
2. **三比值法** —— DL/T 722 §10.2.1 表6/表7(已实现,iec.py)
   - 三对比值(C₂H₂/C₂H₄、CH₄/H₂、C₂H₄/C₂H₆)编码→判定。**方法名:本标准叫「三比值法」,属 DL/T 722,非 IEC**(现有 iec.py 命名名实不符,待校准)。
3. **大卫三角形法(Duval Triangle,或立体图示法)** —— DL/T 722 附录C 图C.2 + 表C.1【国标点名,新增,可视化】
   - ⚠️ **图示法两种,只做三角法**:附录C 图示法含**立体图示法**与**三角图示法(大卫三角)**两种;本系统只采用三角法——1498.2 §5.4.2 点名的是大卫三角、二维分区可视化直观、且立体图示法与三比值法同基于三对比值属同质(避免堆同质方法,同砍 One-Class SVM / 孤立森林之理)。**立体图示法列出但不实现。**
   - 坐标公式(附录C C.1-C.3):X=C₂H₂、Y=C₂H₄、Z=CH₄ 含量,
     %C₂H₂=100X/(X+Y+Z)、%C₂H₄=100Y/(X+Y+Z)、%CH₄=100Z/(X+Y+Z)
   - 分区(表C.1):PD=98%CH₄;D1=23%C₂H₄/13%C₂H₂;D2=23%C₂H₄/13%C₂H₂/38%C₂H₄/29%C₂H₂;T1=4%C₂H₂/10%C₂H₄;T2=4%C₂H₂/10%C₂H₄/50%C₂H₄;T3=15%C₂H₂/50%C₂H₄(PD局放/D1低能放电/D2高能放电/T1热<300℃/T2热300-700℃/T3热>700℃)。图C.2 另有 **D+T 混合区**(放电兼过热)。
   - **前端做三角形分区可视化**(直观、答辩亮眼)。
   - ⚠️ 附录C 是**资料性附录**(推荐非强制);表C.1 数值简略,精确分区边界实现时建议参考 Duval 原始论文定义。

**三方法怎么统一/比一致性(已定,关键)**:三方法分类粒度不同,**不强行统一代码**(国标无表5↔六代码官方映射,硬映射=无依据瞎输出),而是分两层看一致性:
- **三比值法 + 大卫三角**:本就是**同一套六代码**(PD/D1/D2/T1/T2/T3,表B.1 判据表即证),直接比落格是否一致(同格/相邻格→一致);
- **特征气体法(表5)**:保留原话输出(「判定油过热」等),**不硬翻成 T3/D2**(表5 按「油/油+纸」分,粒度与六代码不齐;区分油/纸靠残缺的 CO/CO₂,本就受限);
- **三方一致性统一拉到「放电 vs 过热」性质大类比**:三方是否都指向过热(或都指向放电)——都指→可信度高;打架→标可信度低。

**增值 / 答辩点**:分类粒度不同不是缺陷,恰体现「多方法各有侧重」(三比值/Duval 给六分区细分,特征气体法补「油/纸」维度);零编造、全程国标原文。输出示例:「判定高温过热(高温 T3),三比值与大卫三角一致、特征气体法亦指向油过热——三方一致,可信度高。建议结合附录D推荐的电气试验(绕组直流电阻、铁芯绝缘电阻等)进一步核实故障部位。」

- **输出定位(依 DL/T 722,给结论不自缚)**:
  - ✅ 输出:气体特征(「乙炔类显著」「产热类偏高」)、**故障类型判断结论**(三比值/Duval 六代码 + 特征气体法的过热/放电类型)、方法一致性、**可能成因参考**(按类型列举常见成因,见下「成因参考表」,标注「需停电试验确认、系统不区分」)、措施建议(附录D 进一步试验)。
  - **成因参考表(路线乙,克制列举——依 DL/T 1685 附录B 表B.2/B.3 提炼,仅列举不判断)**:报告加一行「可能成因参考(需停电试验确认,系统不区分):XXX」。映射:高温过热T3→接头松动/铁芯多点接地/油道堵塞/磁屏蔽短路;中低温过热T1/T2→导电回路接触不良/涡流损耗/局部过热;高能放电D2→分接开关拉弧/绕组引线击穿/匝间短路;低能放电D1→悬浮电位接触不良/油流带电/气泡放电;局放PD→金属尖端放电/悬浮放电/绝缘受潮。⚠️ **只列举、不判断具体是哪个**(区分成因需停电试验,本系统只有DGA,判不了——守「类型判断=输出,成因/部位=展望」的边界,不越界确认)。
  - ⚠️ 可信度标注(**非边界,是诚实**):低浓度<10μL/L(§10.2.4 c)或三方打架时,标「可信度低」,但仍给判断结论。
  - ❌ 不做:**健康评分**(不在 DL/T 722 范围内,那是 DL/T 984 老化导则的事,非本系统职责);**运维处置指令**(停电/检修属现场运维决策,超出「分析后台」职责,只给到附录D「进一步试验」层)。
  - 依据:DL/T 722 第1章职责 = 「故障识别 + 故障类型判断 + 建议进一步采取的措施」——判类型、给措施正是国标要求。
- **用国标规则法而非 ML 多分类器(数据事实,诚实交代)**:三比值/Duval/特征气体是**规则查表法,不需训练、不需标签**,直接给类型结论完全合规。之所以不额外训 ML 多分类器,是因为 `bmrayan` 真故障仅 4 条(18 条标签),验证不了 ML 多分类准确率 —— 故用国标规则法。措施建议依据:**DL/T 722 附录D 表D.1「据 DGA 判断有故障后,结合表D.1 推荐的其他电气试验(绕组直流电阻/铁芯绝缘/绝缘油击穿/局放测量等,多需停电)才有助于故障部位判断」**——即 **DGA 后台的职责边界到「故障类型判断(过热/放电性质 + 六分区)」这层**,报告如实给出此结论 + 附录D 措施建议;而**具体成因/物理部位定位**(哪个绕组、哪处接头)需接停电电气试验,超出在线 DGA 能力,如实标注「部位定位需附录D 停电试验」。这不是「不敢确诊类型」,而是「类型已判定,部位待现场试验」——层次分清:**类型判断=系统输出,部位定位=展望**。

### 模块2 预警告警(主线下游,对标附录A)
- **职责**:按 DL/T 1498.2-**2025**(2026-06-18 实施,最新版)**附录A 表A.3**(220kV及以下)出告警等级。**注**:分级动作已在模块1第①步「检测=分级」完成,本模块负责**四档全报的输出组织 + 趋势增量判据并入**;分级表只一处(表A.3),不重复判。
- **两套概念别混**(据国标正文核实):
  - **A.1 信息分类(定性,三类)**:注意信息 / 告警信息 / 停运信息——这是全国标通用的信息严重性分类(含停运,为覆盖特高压)。
  - **表A.3 阈值档位(220kV具体,四档)**:正常 / 注意值1 / 注意值2 / 告警值,**无停运值**(停运只特高压表A.1有)。**本系统按表A.3 走 = 四档,不设停运档。**
- **告警措辞背书(A.1 原文)**:A.1 措辞——注意值「设备**可能存在隐患**,需加强监视」、告警值「**可能存在缺陷、有可能发展为故障**,需采取措施」、停运值「**可能发生严重异常**」。**注**:此处「可能」是**告警等级**层面的措辞(隐患严重性用概率化表达),与**故障类型判断**层是两回事——告警等级说「可能有隐患」,故障类型判断层照 §10 三比值/Duval 给出「高温过热/高能放电」类型结论,二者不矛盾。
- **四档全报**:正常 / 注意值1 / 注意值2 / 告警值,**每条都报**(正常也输出「正常,各气体在范围内」),非「仅异常才报」。
- **告警四档阈值(规范性,直接照国标表A.3,三组判据)**:

  **① 绝对浓度(μL/L)——看当前值多高**
  | 项目 | 正常 | 注意值1 | 注意值2 | 告警值 |
  |---|---|---|---|---|
  | C₂H₂ | <1.0 | ≥1.0 | ≥5.0 | ≥8 |
  | H₂ | <120 | ≥120 | ≥150 | / |
  | 总烃 | <120 | ≥120 | ≥150 | / |

  **② 绝对增量(μL/L·周)——看每周涨多少**(与参比值差,公式A.1;数值据 DL/T 1498.2-2025 表A.3 **原图**核对)
  | 项目 | 正常 | 注意值1 | 注意值2 | 告警值 |
  |---|---|---|---|---|
  | C₂H₂ | <1.0 | 「从无到有」或增量≥1.0 | ≥2.0 | ≥4 |
  | H₂ | <15 | ≥15 | ≥30 | / |
  | 总烃 | <15 | ≥15 | ≥30 | / |

  **③ 相对增长速率(%/周)——看每周涨百分之多少**(公式A.2)
  | 项目 | 正常 | 注意值1 | 注意值2 | 告警值 |
  |---|---|---|---|---|
  | 总烃 | <30 | ≥30 | ≥60 | / | (⚠️ 总烃<30μL/L 不计算相对增长速率)

  → **②③ 是产气速率/增量判据,且本身就分四档**:既作模块1检测的「涨势研判」(超注意值即一票),又作模块3趋势预警的「预」来源。三组判据全在表A.3 一张表,四档口径统一。
- **告警输出三层信息**(从硬到软):
  1. 告警等级(注意值1/2、告警值)← 国标阈值定,硬核输出
  2. 超标气体 + 产气速率 ← 实测事实,输出
  3. 故障类型判断(§10 三比值/Duval 结论)← 输出,不决定告警等级(等级由表A.3阈值定),但给出类型结论 + 措施建议

### 模块3 趋势预警(辅线,服务预警——「预」的来源)
- **职责**:基于合成时序算产气速率,把增量/增长率判据**喂给预警**(涨势异常提前报 = 题目的「预」),并做趋势展示。
- **与模块1「处置紧急度研判」里产气速率的分工**(同一个产气速率,两个用途,不重复):模块1 研判=判「涨势超没超注意值」给 yes/no 供 Agent 定处置紧急度;本模块=完整趋势曲线展示 + 「预」的提前预警逻辑。产气速率正好成为检测与趋势辅线的纽带。

- **国标计算公式(DL/T 722 §9.3.2,实现照此)**:
  - **相对产气速率(式2,推荐——只需前后两次浓度,最易实现)**:
    γᵣ = (C₂ − C₁) / C₁ × 1/Δt × 100%  (%/月);C₂=后次浓度,C₁=前次浓度,Δt=间隔(月)
  - 绝对产气速率(式1,需油量m、油密度ρ,较繁):γₐ = (C₂−C₁)/Δt × m/ρ (mL/天)
  - **总烃相对产气速率注意值 = 10%/月**(总烃起始含量很低<10μL/L 时不宜用此判据,故设「总烃≥10μL/L」门槛,依据 §9.3.2)。
  - 表4 绝对产气速率注意值(mL/d,密封式/开放式,**原图核对**):H₂ 10/5、C₂H₂ 0.2/0.1、总烃 12/6、CO 100/50、CO₂ 200/100。
    - 注:C₂H₂<0.1 且总烃低时总烃产气率可不分析;检测周期缩短时本表仅供参考。

- **「预」的国标合法性依据(关键,答辩引用)** —— DL/T 722 §9.3.3 a):
  > 「气体含量虽**低于注意值**,但**产气速率超过注意值**,也应缩短检测周期(预警)。」
  → 这就是「**预**」:气体还没超标,但涨势异常 → 提前预警。不是靠预测模型,靠产气速率判据,且**有国标明文撑腰**。

- **不做时序预测模型**(ARIMA/LSTM 等一律不做):辅线只做产气速率/增量的**趋势展示 + 判据预警**,不做未来浓度预测。合成数据上论证趋势判据可行即可,不吹精度。

### 模块4 Agent(创新点,轻量)——题眼,承担确定性方法做不到的两件事

> **为什么非 Agent 不可**:模块1-3(分级/处置研判/故障类型判断/趋势)全是**确定性方法**(查表/投票/算数),规则代码就能做,不需要 LLM。Agent 的**不可替代价值**只在「需要判断或表达」处——即下面 B、C 两件事,是 if-else 做不好、必须靠 LLM 的。分析层保证「准」,智能体层保证「智能」。

- **职责**:编排「分级+处置研判 → 故障类型判断 → 趋势 → 预警」流程,并承担两件核心智能任务:

  **B. 生成自然语言分析报告(必做,LLM 表达价值)**
  - 把多维分析结果(档位/超标气体/产气速率/处置紧急度研判/故障类型判断+措施建议)综合成一段**人话报告**。
  - **命名:叫「分析报告」**(合题目「分析智能体」)。报告依 DL/T 722 给结论:检测 + 趋势 + **故障类型判断** + 措施建议;不做健康评分(非 722 范围)、不给运维处置指令(只到附录D 试验层)。若被追问「分析出故障类型了吗」:答「用国标规则法(三比值/Duval)判出故障类型(过热/放电性质+六分区);未训 ML 多分类器因真故障仅4条无法验证准确率;具体成因/部位定位需附录D 停电试验,归展望」。
  - 报告内容:预警级别 + 超标项(实测) + 趋势(产气速率) + 处置紧急度研判(产气速率) + **故障类型判断**(三方融合结论 + 可信度) + 措施建议(见C)。
  - **报告版式 = DL/T 722-2014 附录G 表G.1「油中溶解气体分析档案卡片」**(资料性附录,报告直接套此国标原表版式,答辩背书硬)。字段映射如下,**有数据如实填、无数据诚实标「—/不适用(合成数据)」,不杜撰(P1 诚实原则:合成环境拿不到的字段如实留空,不编造)**:
    | 表G.1 分区 | 字段 | 本系统填法 |
    |---|---|---|
    | 设备铭牌 | 型号/制造厂/出厂序号/油重/油种/冷却方式/调压方式/油保护方式/出厂年月/投运日期 | **「—(合成虚拟设备,无铭牌)」**;仅「电压等级」填「220kV及以下(前提声明)」 |
    | 取样条件 | 年月日时 | 合成时序当日日期,如实填 |
    | 取样条件 | 取样原因/取样部位/油温/负荷 | **「—(合成数据无此工况)」** |
    | 含气量% | — | **「—」** |
    | 组分含量 μL/L | H₂/CH₄/C₂H₄/C₂H₆/C₂H₂ | 当日实测(合成),如实填 |
    | 组分含量 μL/L | CO/CO₂ | 有值则填、缺失标「—」(5主2辅:5烃类核心 + CO/CO₂ 残缺作辅助) |
    | 组分含量 μL/L | **O₂/N₂** | **「—(数据集不含此两组分)」** |
    | 组分含量 μL/L | C₁+C₂(总烃) | 计算填 |
    | 速率区 | 总烃增长(μL/L) | 相邻两次差值,填 |
    | 速率区 | 实际运行时间(天) | 合成时序间隔,填 |
    | 速率区 | **总烃产气率(mL/天)** | **「—(绝对产气率式1需油重/油密度,本系统用相对产气率式2,%/月)」** |
    | 收尾 | 试验报告编号 | **「—」** 或合成批次号 |
    | 收尾 | **分析意见** | **← Agent 生成的自然语言报告落此栏**(级别+超标+趋势+故障类型判断+措施建议;不做健康评分、不给运维处置指令) |
  - ⚠️ 表G.2(其他检查性试验/检修情况/故障记录)**不做**——需真实台账,合成环境无。答辩话术:「报告严格套用 DL/T 722 附录G 表G.1 档案卡片版式,合成环境无法获得的铭牌/工况字段如实留空、不杜撰——既是国标背书,也是 P1 诚实原则的体现」。

  **C. 自主决策下一步动作(做,输出决策建议;体现「智能体」核心 = 有决策)**
  - 规则**全有国标依据,不用编**:
    - **可信度低/有分歧** → 建议二次采样验证(§5.4.5 原文:发现预警后**自动二次采样验证,确认后自动缩短为快速采样周期**——两步:先验证→确认后→再缩周期,Agent C 按此)。
    - **正常状态默认周期**(基线)→ 参 §5.3 表1(220kV 6个月 / 66kV 1年;仅作决策基线引用,非合成数据实际采样频率)。
    - **异常时缩短周期**(参 §5.4 b),按故障类型判断结论触发):
      | 故障类型 + 涨势 | 建议缩短到 |
      |---|---|
      | 过热、疑主磁/漏磁回路 | 每周一次 |
      | 过热、疑导电回路 | 每天一次 |
      | 放电(低能量) | 每天一次 |
      | 放电(高能量) | 建议进一步做附录D 电气试验(局放测量等);不下「退出运行」等运维处置指令 |
  - ⚠️ 只输出**监测动作决策**(二次采样/调周期)+ **附录D 进一步试验建议**(§5.4 b)/附录D 有国标依据),**不输出运维处置指令**(停电/检修属现场运维,超出分析后台职责)。
  - ⚠️ **诚实交代(P1)**:合成环境下只**演示决策产出**(打印建议),不涉真实采样执行;二次采样/调周期在合成数据上无法真正复采,如实说明。

  **D. 自由对话问答 —— 不做**(幻觉/易越界确诊/多轮状态/不可控,坑多且风险高)。想要交互感用「固定按钮(为什么报警/趋势如何)+ B式受控生成」替代,不开放自由问答。

- **轻量原则**(导师明示):不强求复杂自主规划;B+C 两件事跑通、讲清「Agent 不可替代性」即可。
- **职责范围 + 降级**:输出 = 级别+超标气体+故障类型判断+措施建议+监测决策;不做健康评分(非722范围)、不给运维处置指令(只到附录D 试验层);LLM 不稳时保留规则降级(直接用模板出报告)。
- **答辩话术**:「分析层用成熟确定性方法(国标规则法)保证准确;智能体层做两件规则代码做不到的事——① 综合多维结果生成自然语言分析报告;② 据可信度与产气速率自主决策二次采样/调周期(对标§5.4.5)。这是'智能体'的不可替代价值。」报告依 DL/T 722 给出故障类型判断 + 措施建议;成因/部位定位需附录D 停电试验,归展望。

### 模块5 依据知识库 + 角标引用(增值项,⏳ 有余力再做——主线做完,来得及就做,来不及不影响毕设完整性)

> **目标**:让报告「有据可依」——每个结论带角标【1】【2】,点角标展示对应国标原文,定位到标准/章节/页码。**不用 RAG**(判据有限、口径易串,结构化更准更可控)。

- **① 依据知识库(结构化,非RAG)**:存**判据条目**,每条自带精确位置 `{ id, 标准名, 章节/表号, 原文片段, 页码 }`。存哪些:722(表5/表7/表A.3判据/附录D措施)、1498.2(表A.3)、1685(附录B成因);**只存判据,不灌整篇PDF**。定位精度:标准名 + 章节/表号 + 原文 + PDF页码跳转。
- **② 结论↔依据绑定(规则,确定)**:分析引擎每个结论生成时**带出所用判据 id**(如 T3→`722-表7`),这是代码里确定的对应,不需检索。
- **③ Agent 约束式挂角标(B报告的增强)**:把规则给的「结论+依据id」素材喂 Agent,Agent 只负责**组织成人话报告 + 把角标放到正确句子后**;**不许自己发明依据**(依据由规则定死,防幻觉)。—— **依据「是什么」由规则定(准),角标「放报告哪句后」由 Agent 做(活)**,这是 Agent 在本模块的不可替代价值。
- **④ 前端**:报告渲染角标 → 点击 → 展示该条国标原文 + 章节 + 跳 PDF 对应页。
- **与 §五关系**:§五国标依据对照表是本模块知识库的静态雏形;做本模块时把它升级成「可点击的依据条目」。
- ❌ **不做**:RAG 向量检索、自由问答(与 Agent D 不做一致)。

## 六、数据策略(已定:bmrayan 真实快照 + 合成时序)

### 数据分工总纲(地基,据物理事实定死)

> **Agent 全程跑合成 360 天时序**,对每天:分级(表A.3 四档全报)+ 检测处置紧急度研判(产气速率 §9.3.3)+ 故障类型判断 + 趋势。

**原始 743 的角色**:① 合成的**分布原料**(按其真实分布——偏度/相关性合成,见下「合成锚点」);② 答辩**诚实交代**数据来源。**不直接进主流程跑检测。**

- **合成时序(关键)**:原始快照**无时间戳**,为获得时间维度,**基于其真实统计分布合成「单台虚拟变压器 × 360 天」时序数据**(保持原始分布特性:偏度、气体相关性等,bmrayan Analysis 页有真实统计值可引用)。
