## 系统一句话

面向 220kV 及以下油浸式变压器,基于 DGA 的**分析智能体后台软件**(对标国标站端后台):检测(分级)+ 辅助诊断 + 分级告警 + 趋势预警;创新点 = **智能体(Agent)编排** 整个分析—预警流程。

- **题目**:《变压器油中溶解气体分析智能体研究与实现》
- **数据**:全程跑合成 360 天时序(原始 743 快照仅作合成分布原料);5 主烃类 + CO/CO₂ 辅助
- **输出定位**:依 DL/T 722 输出检测分级 + **故障类型判断结论 + 措施建议**(附录D);**不做健康评分**(非 722 范围)、**不给运维处置指令**(只到附录D 试验层),成因/部位定位归展望

## 实施顺序(不锁时间,先夯地基)

1. **数据地基**(难点1,先做):`synthesis.py` 合成脚本(对齐对数正态/相关性矩阵、异常态按真实故障特征造:电气→C₂H₂、热→C₂H₄·C₂H₆、健康75%/异常25%×360天)+ 预处理3步 + `features.py`(删滑窗、加 Duval 坐标)
2. **检测**(易):`detect/` 表A.3 四档分级 + §9.3.3 处置解读(不改档位)+ 两视角佐证(孤立森林/产气速率)
3. **辅助诊断**(中):`diagnose/` 特征气体法 + 三比值 + Duval 三角(原始论文坐标);仅异常档触发
4. **告警**(易):`warning/` 表A.3 三组阈值判档 + 四档全报
5. **Agent**(难点2,题眼):`agent/` LangChain 编排 + B 分析报告 + C 决策建议;守边界 + 降级
6. **前端 + 联调**:`FE/` 检测对比 / Duval 三角可视化 / 告警面板 / 趋势曲线 + 全链路 Demo
7. **收尾**:按重构项目从零重新梳理论文(旧论文梳理弃用)+ 答辩 PPT + Demo 录屏

## 关键约束(实现时守住)

- **算法层与 Web/DB 解耦**:`algorithms/` 输入输出都是 DataFrame,不碰 HTTP/DB
- **数值以国标原图为准**(pdftotext/OCR 易串行,不作数值依据)
- **检测**:表A.3 分级是**事实档位**(如实四档全报);§9.3.3 应用原则只调**处置建议/紧急度,不改档位**
- **故障类型判断**:只对注意值2及以上触发(§10.2.4 a);多方融合(三比值/Duval/特征气体)输出**故障类型判断结论 + 措施建议**,低浓度/三方打架标可信度低但仍给结论
- **不做**:时序预测模型(ARIMA/LSTM 一律不做)、健康评分(非722范围)、运维处置指令(只到附录D)、成因/部位定位(归展望)
