## 第 3-4 页 | 研究背景与意义

### P3：背景 —— 为什么做？

- 变压器是电力系统核心设备，故障后果严重
- 传统运维模式是"事后维修"——出事才知道
- DGA（油中溶解气体分析）是国际公认最有效的在线监测手段
  - 一张图：变压器 → 取油样 → DGA 分析 → 7 种气体浓度
- **核心痛点**：有数据，但缺乏智能分析 + 自动预警能力

### P4：意义 —— 解决什么问题？

- **一句话**：从"事后处理"到"事前预防"
- 三个转变：
  - 被动检修 → 主动预警
  - 人工看数据 → AI 自动决策
  - 经验判断 → 数据+模型驱动
- 附：国内外研究现状简述（2-3 篇核心文献）

## 第 7 页 | 监测指标体系

**三类核心指标：**

| DGA 气体（7种） | 衍生指标 | 运行工况（3项） |
| :-------------: | :------: | :-------------: |
|   H₂ CH₄ C₂H₆   |   总烃   |      油温       |
|    C₂H₄ C₂H₂    | 产气速率 |    负载电流     |
|     CO CO₂      |  三比值  |    环境温度     |

> DGA 气体是预测核心，工况是辅助。其余指标（局放、振动）受限于数据，暂不纳入。

## 第 8-14 页 | 研究内容（7 个模块）

### 模块总览（1 页）

```
    数据采集 ──→ 数据加工 ──→ 异常检测
         ↓            ↓            ↓
    趋势预测 ──→ 预警决策 ──→ LangChain Agent
                                   ↓
                            可视化大屏
```

### P8：模块 1 —— 数据采集

| 做                                   | 不做             |
| ------------------------------------ | ---------------- |
| 下载公开数据集（IEC TC 10，700+ 条） | 复杂演化数据生成 |
| 数据清洗与格式统一                   |                  |
| 使用真实数据，来源可靠可追溯         |                  |

### P9：模块 2 —— 数据加工

| 做                    | 不做         |
| --------------------- | ------------ |
| 气体浓度、总烃、比值  | 复杂统计特征 |
| 产气速率、温升速率 ⭐ | AHP 层次分析 |
| 滑动窗口 + 归一化     |              |

### P10：模块 3 —— 异常检测

| 做（3 种）       | 不做         |
| ---------------- | ------------ |
| 阈值法           | Autoencoder  |
| IEC 三比值法     | 关联异常检测 |
| Isolation Forest |              |

> 对比实验：准确率 / 召回率 / 误报率

### P11：模块 4 —— 趋势预测 ⭐ 核心

```
ARIMA（基线）  vs  LSTM 多输出回归（主力）
  ↓                    ↓
7 种气体独立建模      单模型同时输出 7 个值
```

- 输入：过去 30 天 → 输出：未来 1-3 天（滚动预测）
- 框架：Keras，`Sequential([LSTM(64), Dense(7)])`
- 对比：MAE / RMSE / MAPE
- 兜底：LSTM 不收敛 → 降级纯 ARIMA

### P12：模块 5 —— 预警决策

|      规则引擎      |      四级分级       |  误报控制  |
| :----------------: | :-----------------: | :--------: |
| 硬规则（国标阈值） | 🔴 红色：已超标/24h | 持续性判断 |
| 软规则（预测结果） |   🟠 橙色：1-3 天   |  24h 去重  |
| 组合规则（多指标） |   🟡🟢 黄色/蓝色    |            |

### P13：模块 6 —— LangChain Agent ⭐⭐⭐ 最大亮点

```
Agent = LLM 大脑 + 4 个工具 + Prompt 固定流程

get_today_data → detect_anomaly → lstm_predict → check_warning → 生成通知
```

- 框架：LangChain + ReAct + 通义千问（qwen-turbo）
- 约 170 行核心代码
- 兜底：Agent 不稳定 → 退回纯规则引擎 Pipeline

### P14：模块 7 —— 可视化大屏

- ECharts 监控大屏
- 预警列表 + 预测趋势曲线
- **Agent 思考过程可视化** ⭐（5 步执行状态、Thought、输入输出）
- AI 通知展示
