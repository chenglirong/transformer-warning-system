## 一、ARIMA > LSTM 怎么办?

### 📊 现象分析
```
预期:LSTM(深度学习) > ARIMA(传统统计)
实际:ARIMA > LSTM(平均 MAE:ARIMA 301.2,LSTM 1287.4)
```

### 🤔 为什么会这样?
这是机器学习领域的经典现象,原因有:

**原因 1:数据量不足**
```
LSTM 需要:大样本才能发挥优势
ARIMA 需要:小样本即可稳定
本项目数据:单设备 360 天 → 对 LSTM 是「小数据」
```

**原因 2:时序模式简单**
```
DGA 气体变化:缓慢、单调、弱周期
ARIMA 擅长:线性趋势、简单周期 ✅
LSTM 擅长:复杂非线性模式(在本场景用不上)
```

**原因 3:演化机理平稳**
```
气体浓度变化由设备老化 + 工况缓慢驱动
演化平稳,缺乏 LSTM 擅长的复杂长程依赖
```

**原因 4:LSTM 过参数化**
```
LSTM(64) 对 7 维输入参数过多;ARIMA 参数少,正好匹配
```

### ✅ 论文中的写法(化失败为亮点)

> **5.4 实验结果与分析**
>
> 表 5-X 对比了 ARIMA 与 LSTM 在七种气体上的预测精度。
>
> (插入对比表)
>
> **意外发现**:在本文实验中,ARIMA 模型的整体预测精度(MAE = 301.2)优于 LSTM 模型(MAE = 1287.4)。这一结果与多数文献中「深度学习优于传统方法」的结论存在差异,值得深入分析。
>
> **原因分析**:
>
> (1)**数据规模限制**:本实验采用单设备 360 天时序数据,对于 LSTM 这类参数密集型模型而言数据量偏少,难以充分训练,容易陷入过拟合;而 ARIMA 模型参数少、对小样本鲁棒性强,更能稳定发挥。
>
> (2)**时序模式特征**:DGA 气体浓度的演化具有「渐进性、单调性、弱周期性」特征,相比股票、电力负荷等强非线性时序,更接近 ARIMA 模型擅长的线性自回归模式。深度学习的非线性建模能力在此类场景下未能体现优势。
>
> (3)**变压器气体物理特性**:气体浓度的变化主要由设备老化和运行工况驱动,演化过程相对平稳,缺乏 LSTM 擅长捕捉的复杂时序依赖。
>
> **结论与启示**:本实验表明,**模型选择应与数据特性匹配,而非盲目追求复杂模型**。在数据量有限、时序模式简单的预警场景下,传统统计方法仍具有不可替代的价值。这一发现对实际工业部署具有重要参考意义——简单可靠的 ARIMA 模型在工程落地中往往更为实用。

## 二、IEC 三比值法召回率低怎么办?

### 📊 现象分析
```
预期:IEC 三比值法(国标权威方法)能可靠识别异常
实际:准确率尚可(0.675),但召回率仅 0.407(漏掉近六成真异常)
```
三方法对比(360 天合成,基准 fault_state,91 异常):

| 方法 | 准确率 | 召回率 | 误报率 | F1 |
|------|-------|-------|-------|-----|
| 阈值法(DL/T 722) | 0.606 | 0.780 | 0.454 | 0.500 |
| **IEC 三比值法** | 0.675 | **0.407** | 0.234 | 0.387 |
| Isolation Forest | **0.803** | 0.604 | **0.130** | **0.608** |

### 🤔 为什么会这样?
这是 IEC 三比值法的**已知缺陷**:
```
IEC 三比值法只能识别「典型故障模式」
对于:边界故障 / 早期故障(气体浓度低) / 复合故障
  → 落入「无法识别(Undetermined)」区域
  → 召回率低
```
这是国标方法的客观局限,不是实现问题。

### ✅ 论文中的写法(化失败为亮点)

> **4.4 三种异常检测方法对比分析**
>
> 表 4-X 对比了三种方法的性能指标。
>
> **关键发现**:(1)阈值法召回率较高(0.780)但误报率也高(0.454),覆盖广而精度低;(2)IEC 三比值法准确率尚可(0.675)但**召回率显著偏低(0.407)**,近六成真实异常未被识别;(3)Isolation Forest 综合表现最佳(F1 = 0.608)。
>
> **理论解释**:IEC 三比值法是基于专家经验制定的「硬规则」,其判定空间存在大量「无法识别」区域,对边界故障和早期故障覆盖不足。Duval 等学者〔引文献〕早已指出这一局限性,并提出了「五边形法」等改进方案。
>
> **工程启示**:**单一方法难以兼顾准确率与召回率,多方法协同是预警系统的必然选择**。同时,IEC 召回仅约四成的事实,也构成本系统「只输出二分类预警、不对外给出具体故障类型」边界设定的直接依据——在此召回水平下输出确定性故障诊断既不可靠也不严谨。

## 三、预警 70% 误报率怎么办?

### 📊 现象分析
```
预期:预警误报应尽量低
实际:误报率 70.4%(召回 0.65;混淆 TP=52 FP=176 FN=28 TN=74,330 天回测)
```

### 🤔 为什么会这样?
**原因 1:预警优先召回**
```
为不漏报 → 阈值设保守 → 误报增加(预警领域经典权衡)
```
**原因 2:数据弱可分(根本原因)**
```
CO 正常/异常分布几乎重叠(可分性 d 仅 0.11,正常均值 349 vs 异常 392)
  → 按浓度阈值报警必然大量误报,是判别天花板而非规则缺陷
```
**原因 3:缺上下文**
```
未引入设备年限、负载历史、季节因素等上下文
```
注:误报率已通过「CO 阈值校准 + 连续 N 次持续性 + 24h 去重」从 93% 降到 70%。

### ✅ 论文中的写法(化失败为亮点)

> **6.5 预警决策系统效果评估**
>
> 系统在 330 天回测中召回率达 0.65,误报率为 0.704。70% 的误报率乍看较高,但需结合预警系统的特殊性解读。
>
> **(1)预警 vs 诊断的本质差异**:与故障诊断追求精确分类不同,预警系统的核心目标是「宁可错报,不可漏报」。工业实践中,漏报一次故障的代价(设备损坏、停电事故)远大于多次误报的代价(人工核查)。因此预警系统通常以召回率优先。
>
> **(2)误报来源分析**:经分析,误报主因是 CO 等关键气体在正常态与异常态的浓度分布高度重叠(可分性指标 d 仅 0.11),使基于浓度阈值的判别存在固有天花板。这并非规则引擎缺陷,而是数据可分性的客观约束。
>
> **(3)误报控制的工程权衡**:本文设计了「持续性判断 + 时间窗去重」机制,使误报率从 93% 降至 70%。进一步降低误报需引入更多上下文(设备年限、负载历史)与智能复核,作为未来工作方向。
>
> **结论**:本文如实呈现 70% 的误报率而非通过调参美化,其根源是合成数据弱可分性这一客观约束。揭示并量化该约束,本身即是研究严谨性的体现。

## 四、融合不如 iForest 怎么办?

### 📊 现象分析
```
预期:多方法融合 > 任一单方法
实际:等权融合(0.756)< 最优单方法 iForest(0.803)
```

| 方法 | 准确率 | 召回率 | 误报率 | F1 |
|------|-------|-------|-------|-----|
| Isolation Forest(最优单方法) | **0.803** | 0.604 | **0.130** | **0.608** |
| 等权融合(≥2/3 票) | 0.756 | 0.637 | 0.204 | 0.569 |

### 🤔 为什么会这样?
**原因 1:弱方法拖累强方法**
```
IEC(召回 0.407)、阈值法(误报 0.454)的误判
在等权投票中稀释了 iForest 的正确判断
```
**原因 2:方法间相关性高**
```
三方法都基于 DGA 气体浓度 → 错误模式相似 → 融合无法纠错
```
**原因 3:融合策略简单**
```
等权多数投票未按方法可靠性动态赋权
```

### ✅ 论文中的写法(化失败为亮点)

> **4.5 多方法融合实验**
>
> 本文尝试将三种方法以「≥2 票判异常」的等权多数投票融合,期望通过互补提升性能。然而结果显示,**等权融合的准确率(0.756)、F1(0.569)均不及最优单方法 Isolation Forest(0.803 / 0.608)**,仅召回率略高(0.637 vs 0.604)。
>
> **原因分析**:(1)**弱方法稀释**——IEC(召回 0.407)与阈值法(误报 0.454)的误判削弱了 iForest 的正确判断;(2)**方法同质**——三方法均以 DGA 浓度为输入,错误模式相关性高,融合难以「取长补短」,这与集成学习中「基模型需具备差异性方能提升集成效果」的理论一致〔引文献:Dietterich〕;(3)**策略简单**——等权投票未按方法可靠性赋权。
>
> **结论与系统决策**:在数据驱动方法已表现优异的前提下,**简单等权融合并不能无条件提升性能**。基于此,本文最终采用「**Isolation Forest 主导 + 阈值法/IEC 三比值法辅助**」的主辅协同架构——前者作为核心检测器保证精度,后两者作为补充规则提供白盒可解释性。这一设计将「融合未优」的实验发现转化为系统架构选择的实证依据。

## 五、贯穿性根因:数据弱可分性

上述四个现象中,前三个(检测召回低、预警误报高、融合未优)共享同一根因——**正常/异常气体分布高度重叠**;第四个(ARIMA>LSTM)的根因是小样本。

正常 vs 异常 各气体可分性 d(类 Cohen's d,越小越难分):

| 气体 | 合成数据 d | 原始 743 行 d |
|------|-----------|--------------|
| CO | **0.11** | 0.20 |
| C₂H₂ | 0.29 | **0.00** |
| C₂H₄ | 0.41 | 0.24 |
| H₂ | 0.47 | 0.11 |
| CH₄ | 0.52 | 0.36 |
| C₂H₆ | 0.60 | 0.28 |
| CO₂ | 0.80 | 0.24 |

- 除 CO₂ 外全部高度重叠;异常组总烃中位数仅 90(注意值 150),**66% 异常样本未超国标** → 合成的是早期/轻微潜伏故障(DGA 行业公认难点)。
- **关键反证**:原始数据可分性更低(C₂H₂ d=0.00),合成数据反而略高 → 弱可分是**数据集固有性质,非合成器引入**。故不调合成器(硬调异常显著 = 失真)。
