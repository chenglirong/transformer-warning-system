## 一、ARIMA > LSTM 怎么办?

### 📊 现象分析
```
预期:LSTM(深度学习) > ARIMA(传统统计)
实际:ARIMA > LSTM,但差距不大(总体平均 MAE:ARIMA 148.3,LSTM 160.4;
      逐气体 ARIMA 胜 6/7,仅 CO 一项 LSTM 略优)— D-044 新数据重跑
```

### 🤔 为什么会这样?
这是机器学习领域的经典现象,原因有:

**原因 1:数据量不足**
```
LSTM 需要:大样本才能发挥优势
ARIMA 需要:小样本即可稳定
本项目数据:单设备 360 天 → 对 LSTM 是「小数据」
```

**原因 2:时序模式简单**
```
DGA 气体变化:缓慢、单调、弱周期
ARIMA 擅长:线性趋势、简单周期 ✅
LSTM 擅长:复杂非线性模式(在本场景用不上)
```

**原因 3:演化机理平稳**
```
气体浓度变化由设备老化 + 工况缓慢驱动
演化平稳,缺乏 LSTM 擅长的复杂长程依赖
```

**原因 4:LSTM 过参数化**
```
LSTM(64) 对 7 维输入参数过多;ARIMA 参数少,正好匹配
```

### ✅ 论文中的写法(化失败为亮点)

> **5.4 实验结果与分析**
>
> 表 5-X 对比了 ARIMA 与 LSTM 在七种气体上的预测精度。
>
> (插入对比表)
>
> **意外发现**:在本文实验中,ARIMA 模型的整体预测精度(总体 MAE = 148.3)略优于 LSTM 模型(160.4),逐气体对比 ARIMA 在 7 项中胜 6 项。这一结果与多数文献中「深度学习优于传统方法」的结论存在差异,值得深入分析。
>
> **原因分析**:
>
> (1)**数据规模限制**:本实验采用单设备 360 天时序数据,对于 LSTM 这类参数密集型模型而言数据量偏少,难以充分训练,容易陷入过拟合;而 ARIMA 模型参数少、对小样本鲁棒性强,更能稳定发挥。
>
> (2)**时序模式特征**:DGA 气体浓度的演化具有「渐进性、单调性、弱周期性」特征,相比股票、电力负荷等强非线性时序,更接近 ARIMA 模型擅长的线性自回归模式。深度学习的非线性建模能力在此类场景下未能体现优势。
>
> (3)**变压器气体物理特性**:气体浓度的变化主要由设备老化和运行工况驱动,演化过程相对平稳,缺乏 LSTM 擅长捕捉的复杂时序依赖。
>
> **结论与启示**:本实验表明,**模型选择应与数据特性匹配,而非盲目追求复杂模型**。在数据量有限、时序模式简单的预警场景下,传统统计方法仍具有不可替代的价值。这一发现对实际工业部署具有重要参考意义——简单可靠的 ARIMA 模型在工程落地中往往更为实用。

## 二、三种检测方法的互补性(各有侧重,而非「谁更弱」)

### 📊 现象分析
```
三方法在同一基准(fault_state)上呈现「精准 vs 覆盖 vs 均衡」的互补格局,
而非「某一方法明显失败」
```
三方法对比(360 天合成,基准 fault_state,91 异常;D-044 全链路同口径重跑):

| 方法 | 准确率 | 召回率 | 误报率 | F1 |
|------|-------|-------|-------|-----|
| 阈值法(DL/T 722 表3 注意值) | **0.825** | 0.670 | **0.123** | 0.659 |
| 三比值法(DL/T 722-2014) | 0.583 | **0.813** | 0.494 | 0.497 |
| Isolation Forest | 0.803 | 0.604 | 0.130 | **0.608** |

### 🤔 三者为何如此分布?
```
阈值法(表3):气体绝对值超国标注意值才报
  → 精准(误报仅 0.123),但漏掉「绝对值未超、仅比值异常」的早期故障(召回 0.670)
三比值法(表6编码+表7判断):三对比值落入故障编码区即报
  → 覆盖广、召回高(0.813),但判据宽松、误报高(0.494)
Isolation Forest:无监督,不偏任何规则口径
  → 综合最均衡(F1 0.608)
```
这不是「方法有缺陷」,而是**三种判据的设计取向不同**:阈值法重精度、三比值法重覆盖、iForest 重均衡。

### ✅ 论文中的写法

> **4.4 三种异常检测方法对比分析**
>
> 表 4-X 对比了三种方法的性能指标。
>
> **关键发现**:三种方法呈现清晰的互补格局——(1)**阈值法**(DL/T 722-2014 表3 注意值)精度最高(误报率仅 0.123、准确率 0.825),但因仅判定气体绝对浓度是否超注意值,对「绝对值未超、仅比值异常」的早期故障覆盖不足(召回 0.670);(2)**三比值法**(DL/T 722-2014,表6编码 + 表7判断)召回率最高(0.813),覆盖面广,但判据相对宽松导致误报偏高(0.494);(3)**Isolation Forest** 综合表现最均衡(F1 = 0.608)。
>
> **理论解释**:三比值法依据气体比值落入故障编码区判定,覆盖面广但宽松;阈值法依据绝对浓度,严格但对早期故障不敏感——二者是规则法在「覆盖」与「精度」上的两个极点,iForest 作为数据驱动方法居中。这与 DGA 诊断中「单一判据各有盲区、需多判据互补」的工程共识一致〔引文献:Duval 三比值法与 Duval 三角形系列〕。
>
> **工程启示**:**正因三种方法在精度与覆盖上各有侧重,多方法协同(而非依赖单一判据)是预警系统的合理选择**。需说明的是,本系统对外只输出二分类预警、不对外给出具体故障类型——这是预警系统与诊断系统的**职责划分**(预警负责「是否异常 + 趋势」,诊断负责「是什么故障」),而非源于某一方法能力不足;且三比值法虽召回高,其误报率亦近半(0.494),在此基础上对外输出确定性故障类型同样不够稳健。

## 三、预警 52% 误报率怎么办?

### 📊 现象分析
```
预期:预警误报应尽量低
实际:误报率 52.0%(召回 0.64,F1 0.39;混淆 TP=51 FP=130 FN=29 TN=120,330 天回测;D-046 产气速率门槛锚定国标总烃判据 §9.3.2 后,挡掉低浓度涨幅%伪触发,误报率由 D-045 的 74.4% 大降至 52.0%、召回/F1 同步改善)
```

### 📐 为什么是 330 天回测(而非合成的 360 天)?
```
合成时序共 360 天,但前 30 天要留给 ARIMA 当历史输入,真正能回测的只有后 330 天。
  · n = 360(合成总天数)
  · DEFAULT_LOOKBACK = 30(滑窗回看;检测/预测/预警共用常量)
  · 回测目标日 = range(30, 360) → 第 30~359 天,共 330 天
根因:软规则吃 ARIMA 未来 3 天预测,forecast_arima(hist=df[:t]) 需足够历史才稳定拟合;
      第 0~29 天历史不足,被「预热」吃掉,从第 30 天才开始算 TP/FP/FN/TN。
注:持续性过滤(连续 N 次)是在这 330 天内部判每日有效与否,不再额外扣天数,勿与预热混淆。
```
口径区分(答辩别说混):**360 天 = 数据合成规模**(检测不滑窗,用全量 360 评估);**330 天 = 预警回测样本**(360 − 30 预热)。

### 🤔 为什么会这样?
**原因 1:预警优先召回**
```
为不漏报 → 规则设保守(任一硬/软规则触发即报)→ 误报增加(预警领域经典权衡)
```
**原因 2:预测性误报(软规则)**
```
软规则基于 ARIMA 未来 1-3 天预测值是否超注意值;预测本身有误差
  → 预测偏高时产生「预测性误报」,是预警「提前量」与「精度」的固有权衡
```
**原因 3:早期/轻微异常难判**
```
异常多为早期/轻微潜伏故障:异常组总烃中位 179 虽略超注意值 150,
但仍有 41% 异常样本未超注意值 → 边界样本判定困难
```
**原因 4:缺上下文**
```
未引入设备年限、负载历史、季节因素等上下文
```
注:误报率已通过「连续 N 次持续性 + 24h 去重」控制;CO/CO₂ 改为 §10.2.3.1 比值判据(D-044)后,删除了原 CO 绝对阈值硬规则。

### ✅ 论文中的写法

> **6.5 预警决策系统效果评估**
>
> 系统在 330 天回测中召回率达 0.64,误报率为 0.52(F1 0.39;混淆 TP=51/FP=130/FN=29/TN=120)。该误报率乍看仍偏高,但需结合预警系统的特殊性解读。
>
> **(1)预警 vs 诊断的本质差异**:与故障诊断追求精确分类不同,预警系统的核心目标是「宁可错报,不可漏报」。工业实践中,漏报一次故障的代价(设备损坏、停电事故)远大于多次误报的代价(人工核查)。因此预警系统通常以召回率优先。
>
> **(2)误报来源分析**:误报主要来自两方面——其一,软规则基于 ARIMA 对未来 1-3 天的预测值判定,预测误差会转化为「预测性误报」,这是预警「提前量」与「精度」之间的固有权衡;其二,本数据中异常多为早期/轻微潜伏故障(异常组总烃中位 179、仍有 41% 异常样本未超国标注意值),边界样本本身判定困难。
>
> **(3)误报控制的工程权衡**:本文设计了「持续性判断(连续 N 次)+ 时间窗去重」机制抑制噪声尖峰。进一步降低误报需引入更多上下文(设备年限、负载历史)与智能复核(Agent 二次研判),作为未来工作方向。
>
> **结论**:本文如实呈现 52% 的误报率而非通过调参美化(该值由演进中的 74.4% 经「产气速率门槛锚定国标 §9.3.2」校准而来,详见 D-046,系挡掉低浓度涨幅%伪触发、而非调参降误报)。在「召回优先 + 预测提前量」的预警定位下,该误报水平是可解释的工程权衡;诚实揭示并分析其来源,本身即是研究严谨性的体现。

## 四、多方法融合效果

### 📊 现象分析
```
等权融合(≥2/3 票)F1 0.649,优于 iForest(0.608),与最优单方法阈值法(0.659)相当
```

| 方法 | 准确率 | 召回率 | 误报率 | F1 |
|------|-------|-------|-------|-----|
| 阈值法(表3,最优单方法) | **0.825** | 0.670 | **0.123** | **0.659** |
| 等权融合(≥2/3 票) | 0.817 | 0.670 | 0.134 | 0.649 |
| Isolation Forest | 0.803 | 0.604 | 0.130 | 0.608 |

### 🤔 为什么融合此次有效(对比早期实现)?
```
融合效果取决于基方法质量。三比值法故障归类改用 DL/T 722 表7(召回 0.813)后,
不再是早期 IEC 实现(召回 0.407)那样的「弱方法」拖累投票;
三票各有侧重(阈值法精准 / 三比值法高覆盖 / iForest 均衡)→ ≥2 票互补
→ 融合 F1(0.649)超过 iForest(0.608),接近最优单方法阈值法(0.659)
```

### ✅ 论文中的写法

> **4.5 多方法融合实验**
>
> 本文将三种方法以「≥2 票判异常」的等权多数投票融合。结果显示,**等权融合 F1 达 0.649,优于 Isolation Forest(0.608),与最优单方法阈值法(0.659)相当**,在保持较高召回(0.670)的同时将误报率控制在 0.134。
>
> **分析**:融合效果取决于基方法的质量与差异性〔引文献:Dietterich〕。本系统三种方法在「精度—覆盖」维度上分布互补(阈值法重精度、三比值法重覆盖、iForest 居中),「≥2 票」投票要求至少两种判据一致,既吸收三比值法的高覆盖、又借阈值法/iForest 抑制其高误报,从而获得均衡的综合表现。
>
> **结论与系统决策**:多方法协同在本数据上取得了优于单一数据驱动方法(iForest)的效果。本文采用「**多方法投票 + 白盒规则可解释**」的协同架构——融合提供稳健的二分类判定,阈值法/三比值法的规则判据提供可向运维解释的依据。这一设计将「多判据互补」的实验发现转化为系统架构选择的实证依据。

## 五、数据可分性与残余误判的来源

> **重要修正(D-044)**:早期版本曾以「数据弱可分(d 多<0.5)」作为检测召回低/IEC召回40%/融合未优/预警误报高的**统一根因**。该结论在 ground truth 打标改为「DL/T 722-2014 表3 注意值二分 + 表7 故障细分」后**不再成立**——新打标下数据明显更可分(见下表),且三比值法(表7)召回达 0.813。原「弱可分」很大程度是早期 IEC 打标(将气体绝对值不高、仅比值异常的样本也标为异常)人为造成的分布重叠。现据实修正如下。

正常 vs 异常 各气体可分性 d(类 Cohen's d,越大越易分;基准 = 表3 二分打标):

| 气体 | 合成数据 d(新打标) |
|------|-----------|
| CO | 0.14 |
| H₂ | 0.21 |
| C₂H₂ | 0.43 |
| C₂H₄ | 0.61 |
| CH₄ | 0.69 |
| CO₂ | 0.81 |
| C₂H₆ | **0.96** |

- 7 个气体中 5 个 d≥0.43、C₂H₆ 达 0.96,数据**具备合理可分性**(不再是「高度重叠」)。
- 异常组总烃中位数 **179**(略超注意值 150),但仍有 **41% 异常样本未超注意值** → 异常仍以早期/轻微潜伏故障为主,这是 DGA 早期预警的行业难点,也是残余误判(阈值法漏报、预警误报)的主要来源——**不再归因于「数据不可分」,而是「边界/早期样本固有难判 + 预警预测提前量」**。
- ARIMA>LSTM 的根因仍是小样本(360 天,见第一节),与可分性无关。

2. **不洗白误报**:不把误报重命名为「亚健康」等再剔除变相降误报率。如实写 52%。
3. **不编引用**:〔引文献〕处必须查真实可引文献,查不到就不写。
