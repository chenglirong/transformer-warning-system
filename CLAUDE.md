# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目定位

本科毕业设计,题目:**《变压器油中溶解气体分析智能体研究与实现》**。

面向 220kV 及以下油浸式变压器,基于 DGA(油中溶解气体分析)的**分析智能体后台软件**(对标国标站端后台):检测(分级)+ 辅助诊断 + 分级告警 + 趋势预警。创新点 = **智能体(Agent)编排**整个分析—预警流程。

- **唯一设计蓝图**:根目录 `重构设计-DGA分析智能体后台.md`。
- **数据**:全程跑合成 360 天时序(原始 743 快照仅作合成分布原料);5 主烃类 + CO/CO₂ 辅助。

## 目录约定(重要)

- **`v2/` 是现役主力**,所有开发都在这里。结构见 `v2/README.md`。
- **`V1/` 是旧项目(预测路线,已废弃)**,只做归档参考,不再开发、不要动。
- 旧的 LSTM / 时序预测(ARIMA)路线**一律不做**。

## 系统刚性边界(API / 文档 / 答辩都必须守住)

这是**分析+预警系统,不是确诊/评估系统**。

- **对外可输出**:预警等级、表A.3 四档分级、哪些气体超标、趋势/提前预警、诊断**倾向**(不确诊)。
- **对外禁止输出**:具体故障类型确诊、健康度评分。
- **辅助诊断只对异常档触发**(§10.2.4 a),输出「倾向 XXX」不确诊。

## 关键实现约束

- **算法层与 Web/DB 解耦**:`v2/BE/app/algorithms/` 输入输出都是 DataFrame,不碰 HTTP/DB。
- **检测**:表A.3 分级是**事实档位**(如实四档全报);§9.3.3 应用原则只调处置建议/紧急度,**不改档位**。
- **数值以国标原图(用户截图)为准**;pdftotext/OCR 易串行,不作数值依据。
- **环境**:v2 另建 venv,不装 tensorflow(Py3.11 / Intel Mac)。

## 协作纪律

- **改完代码绝不擅自 `git commit`**,是否/何时提交先问用户。
- commit 用 **Conventional Commits** 前缀(`feat(be)`/`feat(fe)`/`test(be)`/`docs`),不用中文前缀;commit message 与决策日志都要**简短**(~4 行)。
- 前端页面巡检**必须跑起来看真实渲染**,不能只读代码说"对"。
- 用户会主动质疑方向:先给证据强度(强/中/弱)+ 2-3 备选 + 推荐,等用户拍板,不要顺着附和。
