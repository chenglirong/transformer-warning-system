## 项目定位

本科毕业设计,题目:**《变压器油中溶解气体分析智能体研究与实现》**。

面向 220kV 及以下油浸式变压器,基于 DGA 的**分析智能体后台软件**:把行业标准串起来 —— 检测(1498.2 表A.3 四档)+ 产气趋势/涨势预警(722 相对速率)+ 故障类型判断(三比值/Duval/特征气体)+ 措施建议 + **附录 G 分析报告**。创新点 = **智能体(Agent)编排**整个分析流程。

- **唯一设计蓝图**:根目录 `重构设计-DGA分析智能体后台.md`。
- **现役目录**:只动 `v2/`(结构见 `v2/README.md`);决策史 `v2/docs/decisions-log.md`。
- **数据**:合成 360 天时序(原始 743 快照仅作分布原料);**7 气全生成**——表A.3 落档只用乙炔/H₂/总烃,CO/CO₂ 用于表5/CO₂/CO(不作「辅气」,也不假装处处同等进门槛)。
- **`V1/` / 旧 LSTM·ARIMA 预测路线**:废弃,不动。

## 输出定位(跟国标走)

以 `DL/T 722-2014` 为主判据链路,`DL/T 1498.2` 管在线四档,`DL/T 1685` 附录B 映射试验项目。

- **可输出**:表A.3 档位、722 注意值/产气速率、故障类型、附录D/1685 试验建议、表 G.1/G.2 档案卡片。
- **不做**:健康评分、运维处置指令(停电/检修)、成因/部位定位、时序预测。
- **废止 V1 禁区**:「不许暴露故障类型 / 只给倾向」等无国标依据。

## 关键实现约束

- **算法层与 Web/DB 解耦**:`v2/BE/app/algorithms/` 输入输出 DataFrame,不碰 HTTP/DB。
- **两套速率各归各**:① 表A.3(%/周)只落档;② 722 §9.3.2 总烃月环比(%/月)管处置紧急度 / 趋势 / **涨势预警**。绝对式1(mL/天)需油重→报告填「—」。
- **双门槛**:判型 = 注意值2/告警 **或** 速率超(含涨势预警);处置紧急度仍仅注意值2+(涨势快→高 / 暂稳→中 / 仅H₂→低)。
- **Agent**:工作流串工具(非自由 ReAct);**LLM 只写报告人话**,不改档位/类型;失败 → 规则模板。七步顺序见 `v2/README.md`。
- **三比值一般在超注意或有增长趋势后使用**(§10.2.1、§10.2.4 a)。
- **数值以国标原图为准**;pdftotext/OCR 不作数值依据。
- **环境**:v2 自有 venv,不装 tensorflow(Py3.11 / Intel Mac)。

## 协作纪律

- **改完代码绝不擅自 `git commit`**,是否/何时提交先问用户。
- commit 用 **Conventional Commits**(`feat(be)`/`feat(fe)`/`test(be)`/`docs`),不用中文前缀;message 与决策日志都要**简短**。
- 前端巡检**必须跑起来看真实渲染**,不能只读代码说「对」。
- 用户质疑时:**对的坚持(给依据),说错了才改**;不要一追问就改门槛。先证据强弱 + 2–3 备选 + 推荐,等拍板。
- 文案统一用「涨势预警」,禁写单独的「预」。
