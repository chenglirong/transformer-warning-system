# 电力设备健康管控智能体设备预警系统研究与实现

> 预警的核心在于"前瞻性"——通过对监测数据的趋势分析与智能预测，实现从"事后处理"到"事前预防"的转变。
>
> **预测时间窗口说明**：考虑到变压器气体浓度变化通常较缓慢（以天为单位），且本课题重点在于验证"预测+预警"的完整流程而非追求超前预测时长，本文设定预测目标为未来 1-3 天的状态变化。这一时间窗口已能为运维人员提供充足的预警提前量，也符合变压器运维的实际需求。

---

## 一、论文目标

基于大语言模型智能体的电力变压器智能预警系统，具体目标包括：

1. 实现基于 LSTM 多输出回归的变压器七种 DGA 气体同步预测，预测时间窗口为未来 1-3 天
2. 能检测数据中的异常信号和趋势异常
3. 能决策什么时候、什么等级触发预警
4. 能控制误报率，避免"狼来了"
5. 能让 AI 自动执行预警决策流程
6. 能通过监控大屏直观展示预警结果

### 模块 4：趋势预测

**研究内容**：用过去 30 天的 DGA 监测数据，预测未来 1-3 天的七种特征气体浓度。采用"传统方法作基线 + 深度学习作主力"的对比实验设计。

#### 基线模型（对照组）：ARIMA

- 对每种气体独立建模（共训练 7 个 ARIMA），作为传统统计学基线
- 为 LSTM 提供性能参照

#### 深度学习模型（主力）：LSTM 多输出回归

- **输入**：过去 30 天的七种气体浓度时序数据
- **输出**：未来 1 天的七种气体浓度预测值（训练时单步训练）
- **推理**：滚动预测，用预测值回填输入窗，实现 1-3 天多步预测
- **核心设计**：`Sequential([LSTM(64), Dense(7)])`，单模型同时输出 7 个预测值
- **框架**：Keras（TensorFlow），代码简洁、调试友好

#### 七种气体同步建模的科学价值 ⭐

变压器内部不同故障会产生不同气体组合（如 H₂+CH₄ 指示局部放电，C₂H₂+C₂H₄ 指示高温过热）。本文通过共享隐藏层隐式学习气体间耦合关系，更符合故障机理，也是 LSTM 相对 ARIMA（必须独立建模 7 次）的核心优势。

> ### ⚡ 做什么
>
> 1. ARIMA 基线：statsmodels 库，对 7 种气体各跑一次（for 循环）
> 2. LSTM 多输出回归：`Sequential([LSTM(64), Dense(7)])`
> 3. 滑窗造样本：过去 30 天 → 预测第 31 天（单步训练）
> 4. 滚动推理：用预测值回填，迭代 2 次得到第 32、33 天
> 5. 数据归一化：MinMaxScaler，训练后反归一化还原真实值
> 6. 对比实验：ARIMA vs LSTM，MAE/RMSE/MAPE 三指标
> 7. 可视化：各气体 1-3 天预测趋势曲线（7 张子图）、误差分布直方图

**不做：**

- ❌ 不做 Transformer、Informer 等复杂架构
- ❌ 不做长周期多步预测（7/15/30 天），仅做 1-3 天
- ❌ 不做 GRU 对比（LSTM 已足够）
- ❌ 不做置信区间估计
- ❌ 不做 Prophet（ARIMA 作为基线已足够）

**技术兜底方案：**

- 优先用 Keras LSTM，训 3-5 天能跑通就是胜利
- 如果 LSTM 效果不佳，论文改写为"ARIMA vs LSTM 对比实验，实验表明传统方法在该数据集上更稳健"
- 如果 LSTM 完全不收敛，降级为纯 ARIMA 预测，重心转向规则预警与可视化系统

#### 举个例子

**输入**：2023-01-01 至 2023-01-30 的七种气体每日浓度（真实值）

| 日期  | H₂  | CH₄ | C₂H₂ | C₂H₄ | C₂H₆ | CO  | CO₂ |
| ----- | :-: | :-: | :--: | :--: | :--: | :-: | :-: |
| 01-01 | 12  |  5  | 0.5  |  8   |  2   | 50  | 300 |
| 01-02 | 13  |  6  | 0.6  |  9   |  2   | 52  | 310 |
| ...   | ... | ... | ...  | ...  | ...  | ... | ... |
| 01-30 | 15  |  8  | 0.8  |  12  |  3   | 60  | 350 |

**滚动预测过程：**

- 第 1 步：`[真实 Day1-30] → LSTM → 预测 Day31 = H₂:16.2, CH₄:8.5, ...`
- 第 2 步：`[真实 Day2-30 + 预测 Day31] → LSTM → 预测 Day32 = H₂:17.1, ...`
- 第 3 步：`[真实 Day3-30 + 预测 Day31,32] → LSTM → 预测 Day33 = H₂:18.3, ...`

> 注：每次推理只输出 1 天，通过 3 次迭代实现 1-3 天多步预测。训练简单（只需训练单步预测），推理灵活。

**预期成果：**

- LSTM 多输出回归的七种 DGA 气体 1-3 天滚动预测
- ARIMA vs LSTM 对比实验报告（含 7 种气体精度对比表）
- 训练好的 LSTM 模型文件（`.h5`）
- 预测结果可视化图表（7 种气体 3 天预测趋势曲线）

**论文写法提示（未来工作章节）：**

> 本文聚焦于 1-3 天短期预测与 LSTM 基础架构验证，未来可探索 Transformer、Informer 等更先进的时序预测架构，以及更长周期的多步滚动预测。

### 模块 5：预警决策系统

**研究内容**：预警决策 = 从预测结果 → 判断是否报警 + 报几级警 + 如何触发

**研究核心**：什么条件下报警？报几级？怎么减少误报？报给谁？

#### 1. 预警规则引擎设计 ⭐

| 规则类型 | 说明           | 举例                         |
| -------- | -------------- | ---------------------------- |
| 硬规则   | 基于国标阈值   | C₂H₂ > 5ppm 立即报警         |
| 软规则   | 基于预测结果   | 预测 1-3 天内超标 → 预警     |
| 组合规则 | 基于多指标关联 | 气体涨快 + 温度高 → 综合预警 |

做成可配置的引擎，可以灵活调整。

#### 2. 四级预警分级机制 ⭐

| 等级            | 触发条件               | 响应要求  |
| --------------- | ---------------------- | --------- |
| 🔴 红色（紧急） | 已超标 或 24h 内必超标 | 立即响应  |
| 🟠 橙色（重要） | 1-3 天内可能超标       | 24 小时内 |
| 🟡 黄色（一般） | 趋势异常上升           | 加强监测  |
| 🔵 蓝色（提示） | 轻微波动               | 日常关注  |

#### 3. 误报率控制（规则层面）

- 持续性判断：异常需连续 N 次触发才报警（过滤噪声尖峰）
- 预警去重：同一预警 24 小时内不重复推送

#### 4. 预警触发与推送机制

- 存入数据库（warning 表）
- 前端实时展示（模块 7 大屏）

> ### ⚡ 做什么（实施重点）
>
> 1. 预警规则引擎（if-then 规则，Python 字典实现）
> 2. 四级预警分级（红橙黄蓝）
> 3. 规则引擎实现：规则加载、规则触发、等级综合判定
> 4. 误报率控制：持续性判断 + 预警去重
> 5. 规则验证：历史数据回测、有效性分析
> 6. 预警推送：存数据库 + 前端展示

- ❌ 不做动态阈值学习（写进未来工作）
- ❌ 不做多模型投票（写进未来工作）
- ❌ 不做置信度评估（写进未来工作）

### 模块 6：基于 LangChain 的预警智能体（Agent）

**模块 6 核心定位（三句话）：**

1. **Agent 角色**：基于 LangChain + ReAct 推理模式实现"数据→检测→预测→决策→通知"的自动化预警流程
2. **LLM 角色**：通义千问作为 Agent 大脑，通过 Prompt 引导按固定流程执行，不做开放式自主决策
3. **设计原则**：4 个核心工具 + Prompt 固定流程引导，Agent 卡死时降级为纯规则引擎

**研究内容**：基于 LangChain 框架构建变压器预警智能体，通过工具调用和 ReAct 推理，将 LSTM 预测、异常检测、规则引擎、通知生成整合为端到端的自动化预警决策流程。

#### Agent 核心组成

- **LLM 大脑**：通义千问（qwen-turbo），通过 LangChain ChatTongyi 接入
- **工具集（4 个）**：
  - `get_today_data()` —— 查询最新监测数据
  - `detect_anomaly()` —— 调用异常检测（阈值法 + 三比值法 + Isolation Forest）
  - `lstm_predict()` —— 调用 LSTM 预测未来 1-3 天
  - `check_warning()` —— 调用规则引擎判定预警等级
- **ReAct 推理模式**：思考(Thought) → 行动(Action) → 观察(Observation) 循环
- **Prompt 模板**：固定流程引导，Agent 按预设步骤执行
- **AgentExecutor**：LangChain 执行引擎

#### Agent 工作流程（Prompt 中写死的 5 步）

```
触发：定时任务（每天定时执行一次）
        ↓
Step 1：调用 get_today_data 获取最新数据
        ↓
Step 2：调用 detect_anomaly 进行异常检测
        ↓
Step 3：调用 lstm_predict 预测未来 1-3 天趋势
        ↓
Step 4：调用 check_warning 进行预警判定
        ↓
Step 5：LLM 基于结果生成预警通知文本 + 复核意见
        ↓
通知入库 + 前端展示
```

#### Prompt 模板示例（节选）

```
你是变压器预警专家。请严格按以下 5 步执行：

Step 1: 调用 get_today_data 获取最新数据
Step 2: 调用 detect_anomaly 进行异常检测
Step 3: 调用 lstm_predict 预测未来 1-3 天
Step 4: 调用 check_warning 判定预警等级
Step 5: 综合结果，按标准格式生成预警通知

注意：必须按顺序执行，不要跳步骤。

通知格式模板：

🚨 【{level}预警】{device_name}
📊 当前状态：...
🔮 预测趋势：...
⚖️ 触发规则：...
```

> ### ⚡ 做什么
>
> 1. 基于 LangChain 搭建 Agent（先跑通 hello world，确认环境可用）
> 2. 封装 4 个核心工具，通知生成不单独封装
> 3. 设计 ReAct 模式 Prompt 模板（写死执行步骤）
> 4. 实现 Agent 自动化执行（AgentExecutor + 定时调度）
> 5. 异常处理：Agent 调用失败时降级为纯规则引擎 Pipeline
> 6. 前后端对接：通知文本存入数据库，前端模块 7 展示

**代码量预估：**

| 部分                                                |    行数     |
| --------------------------------------------------- | :---------: |
| 4 个 Tool 封装（@tool decorator + pydantic schema） |   ~80 行    |
| Prompt 模板                                         |   ~30 行    |
| Agent 初始化与执行                                  |   ~40 行    |
| 异常降级处理                                        |   ~20 行    |
| **总计**                                            | **~170 行** |

**不做：**

- ❌ 不单独封装"通知生成工具"（由 LLM 在 Final Answer 中直接输出）
- ❌ 不做自由的 ReAct 推理
- ❌ 不做复杂的 Memory 管理、会话管理
- ❌ 不做多 Agent 协作
- ❌ 不引入 RAG 知识库增强
  > LLM 仅承担"复核"和"通知生成"，规则引擎已提供完整领域知识。RAG 可作为未来工作。

**技术兜底方案：**

- 优先用 LangChain + ReAct Agent，第 12 周前必须跑通基础 demo
- 如果 Agent 不稳定，通过 Prompt 加强流程约束
- **极端情况**：砍掉 LangChain，退回 Pipeline + API 调用

> 本文基于 LangChain Agent 实现预警决策自动化，未来可探索 Agent 自主决策等更高级范式。

## 四、创新点

### 创新点 1：变压器多方法异常检测对比研究

实现阈值法、IEC 三比值法、Isolation Forest 三种方法的系统对比，给出各方法在变压器预警场景下的适用条件与性能差异。

### 创新点 2：基于 LSTM 多输出回归的七种 DGA 气体同步预测 ⭐

区别于传统对每种气体独立建模的方法，本文采用单个 LSTM 模型同时预测七种气体浓度。通过共享隐藏层隐式学习气体间的物理化学耦合关系（如 H₂+CH₄ 指示局部放电、C₂H₂+C₂H₄ 指示高温过热），更符合变压器故障的实际机理，也提升了预测精度与模型效率。

### 创新点 3：基于 LangChain Agent 的预警决策智能化 ⭐⭐⭐ 最大创新

本文基于 LangChain 框架构建变压器预警智能体，通过工具调用机制和 ReAct 推理模式，实现了"数据→异常检测→预测→决策→通知"的端到端自动化预警流程。Agent 严格聚焦于预警边界（4 个工具 + Prompt 固定流程引导），不越界做评估或诊断，体现了清晰的研究定位与务实的工程实现。

## 六、实施期修订（2026-06-03 起）

> 本节记录正式开发阶段（第 6 周起）相对于上述初版规划的修订。修订源于真实数据集情况与工程实操中的发现，**不改变论文的核心目标、创新点与系统架构**，仅在数据来源、字段范围、技术细节层面做务实调整。详细决策见 `docs/05-decisions-log.md`。

### 修订 1：数据库 MySQL → SQLite（技术选型）

- **背景**：评估部署成本与风险
- **修订**：将"业务存储"由 MySQL 改为 SQLite
- **理由**：本项目数据量级（万行）下 SQLite 完全胜任；零安装、零运维、答辩演示风险更低
- **代码兼容**：通过 SQLAlchemy ORM 编写，可平滑迁移至 MySQL
- **影响**：仅影响第 33 行选型表，不影响系统架构与功能

### 修订 2：数据集为快照型 → 引入"分布保持的时序构造"（替代原"不做状态演化数据生成"）

- **背景**：选定的公开数据集（`FinalDataSet_DGA.xlsx`，743 行）为快照型，无时间戳；无法直接用于 LSTM 时序预测
- **修订**：
  - 原模块 1 中"❌ 不做复杂的状态演化数据生成"调整为：**基于真实样本分布特征，通过分布保持的轻量扰动构造时序序列**
  - 本质是"给真实快照样本附加时间维度"，而非"凭空生成数据"
  - 是数据增强、半合成数据集的常见做法
- **论文表述**：在数据章节坦诚说明数据来源与处理方法，重申本课题定位为"验证预测+预警完整流程"
- **影响**：模块 1、模块 4 的数据来源说明需相应调整

### 修订 3：原始 `Fault` 列不可作为 ground truth → 用 IEC 60599 自动打标作为参照

- **背景**：EDA 发现 743 行中 **97.6% 的 `Fault` 列为 `'NA'`**（未标注），有效标签仅 18 行，无法支撑统计意义上的算法评估
- **修订**：
  - **采用 IEC 60599 国际标准三比值法对全量样本进行自动诊断**，得到"参考标签"
  - 阈值法、Isolation Forest 等其他异常检测方法以 IEC 自动诊断结果为基准做横向对比，评估各方法**与国际标准的一致性与互补性**
  - 该思路并非循环论证：被评估的是"阈值法/IF"，IEC 是国际标准；以标准为参照评估非标准方法，方法论成立
- **EDA 实测结果**：
  - 异常样本 261（35.1%）+ 正常样本 229（30.8%）→ ground truth 充足
  - Undetermined 252（33.9%）→ 反映 IEC 三比值法的固有局限，可作论文中"单一方法局限性"的论据
  - 与原始有限的 18 个标签做交叉验证，整体一致
- **影响**：模块 3、模块 5 的算法评估方法论调整为"以 IEC 国标为参照"

### 修订 4：油温/负载/环境温度等工况数据 → 模拟生成

- **背景**：选定的公开数据集仅含 DGA 气体浓度，**不含工况数据**（论文模块 2 要求的油温、温升速率、负载、环境温度均缺失）
- **修订**：
  - 油温：基础温度 + 日波动 + 负载相关项
  - 负载电流:工作日/周末模式 + 正弦周期波动
  - 环境温度：季节性正弦 + 日内波动
  - 上述模拟参数取符合工程经验的合理范围
- **论文表述**：在数据章节明确说明工况数据为模拟生成，及其参数依据
- **影响**：模块 2 特征工程的工况相关项基于模拟数据展开

### 修订 5：H₂O 列删除

- **背景**：EDA 发现 H₂O 列 100% 缺失
- **修订**:从特征列中剔除
- **影响**:无（H₂O 不在论文要求的 7 种核心 DGA 气体内）

### 修订 6:CO/CO₂ 部分缺失的处理预案

- **背景**：CO 缺失 21.8%、CO₂ 缺失 31.4%（其余 5 种核心气体完整）
- **修订**:在时序合成阶段为所有 7 种气体生成完整序列；保留"7 气体同步预测"的论文叙事
- **降级预案**:若合成质量不足导致 LSTM 训练 CO/CO₂ 困难,可降级为"5 气体同步 LSTM + CO/CO₂ 独立 ARIMA",论文叙事相应改为"以 5 气体同步预测为主,CO/CO₂ 独立建模"
- **影响**：模块 4 视实验结果决定最终预测目标的气体数量
